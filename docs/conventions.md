# tide 개발 규약 (PhotonTCP)

이 문서는 tide 개발 사이클을 따르기 위한 프로젝트 로컬 규약을 기록한다.
마일스톤·보고서의 **형식 원본**은 이 문서가 아니라 tide 플러그인 각 스킬에 동봉된
`template.md` 가 단일 원본(single source of truth)이다. 여기서는 형식을 재정의하지 않는다.

## 단계(phase)별 금지 행위

tide 사이클은 단계로 나뉘며 각 단계에서 허용/금지되는 행위가 다르다.

- **milestone**: 다음 작업 단위를 문서로 정의한다. 코드 구현·git 작업 없음.
- **impl**: 마일스톤대로 구현 + 테스트 + 완료 보고서 작성.
  - 🚫 **git 작업 금지** (add/commit/tag/push 등 일절 하지 않는다).
- **review**: 최근 구현을 비판적으로 리뷰 + 리뷰 보고서 + 릴리즈 판정.
  - 🚫 **git 작업 금지** (커밋·태그·푸시 없음).
- **release**: 프리플라이트 → 버전 범프 → CHANGELOG → commit → tag → push.
  - ✅ git 작업은 **release 단계에서만** 수행한다.

> git 작업 차단 가드(tide-guard)는 tide 플러그인이 hook으로 직접 제공한다.
> 프로젝트별 hook 설치는 필요 없으며, 별도 설치 절차를 만들지 않는다.

## 상태 파일 규약

- `.tide/phase` : 현재 사이클 단계를 담는 **로컬 상태 파일**. 커밋 대상이 아니다(`.gitignore` 처리됨).
- `.tide/deps`  : 레포 간 의존을 **한 줄에 형제 레포명 하나씩** 선언하는 **선택·옵트인** 매니페스트.
  - 선언이므로 **커밋 대상**이다(`.gitignore` 처리하지 않는다).
  - 선언 시 fleet이 위상정렬해 권장 처리 순서를 제시한다. 없으면 의존 0(기본 동작).
  - kickoff가 이 파일을 자동 생성하지는 않는다.

## 태스크 ID · 의존성 표기

- 마일스톤 내 작업 항목은 안정적인 **태스크 ID**로 식별한다 (예: `T1`, `T2`, ...).
- 작업 간 의존은 `(deps: T1, T2)` 형태로 항목에 표기한다. 의존 표기가 있으면
  impl/cycle이 이를 근거로 병렬/순차 실행 순서를 정한다.

## 버전 파일 위치

- 버전 단일 원본: **`pyproject.toml`** 의 `[project].version`.
- release 단계의 버전 범프·CHANGELOG 갱신은 이 파일을 기준으로 한다.
- 현재 버전: `0.1.0` (pre-release, 미발행).

## 디렉터리

- `docs/milestones/` : 마일스톤 문서.
- `docs/reports/`    : impl 완료 보고서 · review 리뷰 보고서.
- `docs/project-context.md` : (진행 중 프로젝트에서만) 코드베이스 개요 캐시.
  신규 프로젝트에서는 생성하지 않으며, 코드가 쌓이면 이후 보강한다.

## 멀티 레포 / 대상 레포 규약

- 모든 tide 산출물(골격, `docs/project-context.md`, `.gitignore`, `CHANGELOG.md`, 마일스톤·
  보고서 등)은 **대상 레포 루트**에 만든다.
- **기본**: 대상 레포 = 세션 레포(현행 단일 레포 동작 그대로).
- **상위 폴더 단일 세션**에서 특정 자식 레포를 지시받으면, 그 **자식 레포 루트**가 대상 레포가 된다.
- 격리 원칙: 한 사이클의 산출물은 다른 레포 루트를 침범하지 않는다.

### 멀티 레포 오케스트레이션 (발견)

fleet 계열 스킬이 자식 tide 레포를 발견할 때의 판별 규칙:

- **세는 대상**: 현재 세션 위치(cwd)의 **직속 1단계 하위 디렉터리**.
- **자식 tide 레포 판별**: 해당 하위 디렉터리가 tide 골격(예: `docs/conventions.md` 및
  `docs/milestones/`·`docs/reports/` 구조)을 갖추고 있으면 자식 tide 레포로 본다.
- **제외**: 이름이 `.` 으로 시작하는 숨김 디렉터리는 세지 않는다.
- 자식 tide 레포가 **2개 이상**일 때만 교차 집계(fleet)가 의미를 갖는다.
  2개 미만(0·1개)이면 일반 단일 레포로 동작한다.
