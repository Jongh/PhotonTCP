# M9 — 실물 광학 링크 하드닝 + 검증 (v1.0 후보 확정)

## 목표

M8이 인메모리로만 검증한 `OpticalChannel`의 **실 카메라 캡처 경로**를 실제로 동작·검증 가능한 상태로
끌어올린다. cv2 디바이스의 프레임 신선도·디스플레이 페이싱·파라미터 가드를 보강해 실 광 링크에서
프레임 누락·지연 누적을 줄이고, **한 머신 셀프체크**(화면→자기 카메라, 반이중)와 **2-머신/외부 카메라
실 왕복** 하니스 + 결정적 검증 절차를 제공한다. 이 마일스톤이 끝나면 "빛만으로 실 통신"을 실제
하드웨어에서 재현·검증할 수 있고(수동 사인오프), 그 결과로 **v1.0 후보가 확정**된다. 코드/CI로 검증
가능한 부분(페이싱·버퍼 드레인·가드·dedup 견고성)은 인메모리 결정적 테스트로 못박고, 하드웨어
동작은 스크립트화된 pass/fail 하니스 + 문서화된 수동 절차로 검증한다.

## 배경

M8-review가 v1.0 승격을 막는 잔여로 명시한 항목들이 출처다:

- **M8-review 권장 1 (v1.0 핵심)**: `Cv2Camera` 실 캡처 경로가 테스트·예제 어디에도 실행되지 않음 —
  예제 `--real`은 `Cv2Display`(표시 절반)만 배선하고 캡처는 인메모리, `--camera` 인자는 미사용. 실 카메라
  왕복이 완전 미검증.
- **M8-review 권장 2**: `Cv2Camera`의 `VideoCapture` 내부 버퍼 staleness — QR 디코드가 느려 캡처 루프가
  카메라 FPS보다 느리면 `cap.read()`가 오래된 프레임을 반환해 지연이 누적.
- **M8-review 권장 3**: dedup이 직전 1개(`last_delivered`)만 비교 — nonce 1바이트(mod 256) 래핑과
  맞물려 캡처 누락이 심한 실물에서 이론적 거짓 dedup 코너.
- **M8-review 사소 4**: `poll_interval=0` 시 캡처 루프 busy-spin(하한 가드 필요).
- **M8 자체의 페이싱 부재**: `OpticalChannel.send_frame`은 즉시 표시만 하고 **프레임 hold가 없어**, 실
  카메라가 한 프레임을 잡기 전에 다음 프레임으로 덮이면 누락된다(현재는 ARQ 재전송에만 의존). 실 링크
  신뢰성을 위해 표시 프레임을 카메라가 최소 1회 잡을 만큼 유지하는 페이싱이 필요.

설계 원칙 유지: 코덱 순수성·`Channel` 계약 불변, 상위 계층 무수정. 실물 의존부는 cv2 디바이스/하니스
뒤로 격리하고, 채널 로직 변경(페이싱·dedup·가드)은 **인메모리 페이크로 결정적으로 테스트**한다. 기존
M1~M8 테스트 **회귀 0**이 절대 조건. 하드웨어 동작은 CI에서 자동 검증 불가이므로, 검증은 (1) 스크립트화된
실물 하니스의 pass/fail 출력 + (2) 문서화된 수동 절차 + (3) review 단계의 수동 사인오프로 갈음한다.

## 태스크 상세

### M9-T01: `Cv2Camera` 신선 프레임 보장 (`photontcp/optical/cv2_devices.py`)

- **변경 파일**: `photontcp/optical/cv2_devices.py`
- **버퍼 staleness 해소(권장 2)**: 생성자에서 `cap.set(cv2.CAP_PROP_BUFFERSIZE, 1)` 시도(백엔드가 무시할
  수 있으므로 best-effort — 반환값 검사·실패 무해). `read`에 **드레인 모드** 추가: 가장 최신 프레임을 얻기
  위해 짧게 연속 `grab()` 후 마지막을 `retrieve()`(또는 직전 캐시 프레임과 비교해 최신까지 폐기)하는
  옵션 `drain_to_latest: bool = True`. 드레인은 비차단 한도(예: 최대 K회 grab 또는 grab이 더 이상 새
  프레임을 안 줄 때까지)로 제한해 무한 루프·과지연을 막는다.
- **해상도 옵션(선택)**: `width`/`height` 생성자 인자로 `CAP_PROP_FRAME_WIDTH/HEIGHT` 설정(QR 모듈
  해상도 확보). best-effort.
- 기존 동작 호환: 새 인자는 기본값. `read`는 여전히 BGR `ndarray` 또는 `None` 반환(드레인이 새 프레임을
  못 얻으면 `None`). import는 여전히 하드웨어·창 비접촉. 기존 import 스모크 회귀 없음.

### M9-T02: 디스플레이 페이싱(hold) + 파라미터 하한 가드 (`photontcp/optical/channel.py`, `photontcp/optical/devices.py`)

- **변경 파일**: `photontcp/optical/channel.py`, `photontcp/optical/devices.py`
- **프레임 hold/페이싱(실물 신뢰성)**: `OpticalChannel.__init__`에 `hold: float = 0.0`(초) 추가 —
  `send_frame`이 한 프레임을 표시한 뒤 다음 표시까지 최소 `hold`초가 지나도록 보장(연속 `send_frame`이
  카메라 캡처보다 빨라 프레임을 덮어쓰는 것을 완화). 구현은 **단조 시계 기반 최소 간격**(직전 표시
  시각 기록 후 부족분만 대기). 인메모리·`hold=0`(기본)에선 대기 0이라 기존 동작·테스트와 바이트 동일.
  실 데모/하니스는 `hold`를 카메라 프레임주기 이상으로 설정. (송신은 caller 스레드라 짧은 실 sleep
  허용 — 채널은 이미 실 스레드·실시간 경계.)
- **파라미터 하한 가드(사소 4)**: `poll_interval`이 0 이하이면 작은 양수 하한(예: 1e-3)으로 클램프하거나
  `ValueError` — busy-spin 방지. `MemoryCamera.read`도 `timeout`이 0/None일 때 busy-spin하지 않도록 작은
  하한을 두거나 문서화(결정성 보존 범위에서). 선택을 명확히 하고 docstring 갱신.
- 기존 동작 호환: `hold` 기본 0, poll_interval 기본 0.01. 기존 광학 테스트(`test_optical_channel.py`·
  `test_optical_concurrency.py`) 회귀 없음.

### M9-T03: dedup 견고성 (`photontcp/optical/channel.py`) (deps: M9-T02)

- **변경 파일**: `photontcp/optical/channel.py`
- **nonce 폭 확대(권장 3)**: 송신 nonce를 1바이트→**다중 바이트(예: 2~4바이트, 고정폭 빅엔디언)**로
  넓혀 래핑 주기를 실질적으로 제거. `channel_frame = nonce_bytes + packet`, 수신측은 고정폭 nonce를 떼고
  나머지를 패킷으로 인도. 프레임 오버헤드는 수 바이트 증가(무시 가능).
- **dedup 깊이(선택 보강)**: `last_delivered` 단일 슬롯 대신 **최근 N개 channel_frame의 작은 집합/deque**로
  비교해, 실물 캡처 순서 흔들림(A를 다시 잠깐 캡처 등)에도 견고하게. N은 작게(예: 4~8) 고정. 단일
  슬롯으로도 정확하던 인메모리 경로의 동작은 보존(연속 동일 프레임 1회 인도·연속 동일 패킷 각각 인도).
- 기존 광학 테스트 회귀 없음(특히 "연속 동일 패킷 각각 인도"·"재캡처 1회 인도" 불변 유지). nonce 폭
  변경에 맞춰 테스트의 페이로드/검증이 nonce 길이를 가정하지 않는지 확인.

### M9-T04: 실물 셀프체크 하니스 (`examples/optical_selfcheck.py`) (deps: M9-T01, M9-T02)

- **변경 파일**: `examples/optical_selfcheck.py`(추가)
- 한 머신에서 **반이중 실 광학 경로**를 검증하는 수동 실행 하니스: `Cv2Display`로 알려진 일련의 QR
  프레임(고유 페이로드 다수)을 표시하고, **사용자가 그 화면을 비추도록 둔 `Cv2Camera`**로 캡처해
  `decode_frame`으로 복원, 표시한 페이로드 집합과 비교해 **수신율·정확도 pass/fail**을 출력한다(예:
  "N개 중 M개 정확 복원, 손실 L, 오류 0"). `--camera`/`--window`/`--scale`/`--hold`/`--count` 인자.
- 표시·캡처를 페이싱(`hold`)·드레인(M9-T01)으로 정렬해 실 캡처가 가능한 한 많이 성공하도록. 디코드
  실패는 손실로 집계(오류 아님). 종료 시 cv2 자원 해제.
- 기존 예제 스타일(`_REPO_ROOT` shim·의존성 체크·English 출력·`main()->int`) 일치. CI 미실행(하드웨어
  필요) — 의존성/카메라 부재 시 명확한 안내 후 비0 아님(graceful) 종료.

### M9-T05: 2-머신/외부 카메라 실 왕복 데모 (`examples/optical_link.py`) (deps: M9-T01, M9-T02)

- **변경 파일**: `examples/optical_link.py`
- `--real`을 **실제 `Cv2Camera` 캡처까지 배선**: 단일 프로세스 자기 루프가 불가능한 제약(M8 명시)을
  고려해 **역할 플래그**로 두 프로세스를 구성 — 예: `--role sender`(이 피어는 `Cv2Display`로 표시 +
  `Cv2Camera`로 상대 화면 캡처)와 `--role receiver`(반대). 각 피어는 `OpticalChannel(display=Cv2Display(...),
  camera=Cv2Camera(...), hold=...)`로 구성해 **실물 디바이스 위에서 동일 세션 드라이브**(핸드셰이크→메시지
  →close)를 수행한다. 두 머신(또는 한 머신 화면을 다른 머신 카메라로) 마주보게 배치.
- 기존 기본(인메모리) 모드는 보존(인자 없음). `--real --role …`만 실물. 한 머신 단독 실행 시의 안내·제약
  (화면2·카메라2 또는 외부 카메라)을 docstring·help에 정직하게 유지·갱신. cv2/카메라 부재 graceful 처리.
- 기본 인메모리 모드 회귀 없음(여전히 `python examples/optical_link.py`가 exit 0).

## 완료 기준

1. **신선 프레임(권장 2)**: `Cv2Camera`가 `CAP_PROP_BUFFERSIZE=1`(best-effort) + 드레인으로 최신 프레임을
   반환하며, 드레인은 유한 한도로 과지연·무한 루프가 없다. import는 여전히 하드웨어 비접촉.
2. **디스플레이 페이싱**: `OpticalChannel(hold=…)`가 연속 `send_frame` 사이 최소 `hold`초 간격을 보장하고,
   `hold=0`(기본)에선 대기 없이 기존 동작과 동일하다(인메모리 테스트로 결정적 검증).
3. **파라미터 가드(사소 4)**: `poll_interval` 0/음수 입력이 busy-spin을 일으키지 않는다(클램프 또는
   거부). 동작이 테스트로 확인된다.
4. **dedup 견고성(권장 3)**: 다중 바이트 nonce로 래핑 거짓 dedup이 제거되고, 최근 N 프레임 dedup이
   재캡처/순서 흔들림에서 "1회 인도"·"연속 동일 패킷 각각 인도"를 유지한다(결정적 테스트).
5. **실물 셀프체크 하니스**: `examples/optical_selfcheck.py`가 실 화면→카메라 반이중 경로의 수신율·정확도
   pass/fail을 출력한다(수동 실행). 의존성/카메라 부재는 graceful 안내.
6. **2-머신 실 왕복 데모**: `examples/optical_link.py --real --role …`가 실제 `Cv2Display`+`Cv2Camera`로
   `OpticalChannel`을 구성해 두 피어 세션 드라이브를 수행한다. 기본 인메모리 모드는 회귀 없이 exit 0.
7. **상위 계층 무수정**: 세션·신뢰성·앱·코덱은 변경 없이 보강된 채널/디바이스 위에서 동작한다.
8. **실물 검증 절차 + v1.0 게이트 문서화**: 셀프체크·2-머신 실행법과 v1.0 사인오프 기준이 문서에 명시된다.
9. 모든 변경이 코덱 순수성·`Channel` 계약을 유지하고, **`pytest`로 M1~M9 테스트가 전부 통과**한다(M1~M8
   회귀 0). 하드웨어 의존 동작은 CI 미검증이며 셀프체크·2-머신 하니스의 수동 결과로 갈음한다.
