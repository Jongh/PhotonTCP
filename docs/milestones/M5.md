# M5 — QR 코덱 실물 통합 (이미지 루프백)

## 목표

PhotonTCP 패킷이 **실제 QR 코드 이미지로 인코딩/디코딩되어** 채널을 통과한다. 패킷 바이트를
base64로 감싸 segno로 QR 이미지(numpy 그레이스케일 배열)를 생성하고, OpenCV `QRCodeDetector`로
디코딩해 원본 바이트를 복원한다. 이 이미지를 메모리로 주고받는 `ImageLoopbackChannel`을 만들어,
기존 세션·신뢰성(ARQ)·스트림·채팅 스택이 **수정 없이 QR 이미지 위에서 그대로 동작**함을 검증한다.
이 마일스톤이 끝나면 "패킷 → QR → 이미지 → 디코드 → 패킷" 경로가 실물 라이브러리로 통과하며,
실제 화면/카메라(M6) 직전의 마지막 메모리 단계가 완성된다.

## 배경

로드맵 4단계 "QR 코덱 실물 통합 — 패킷이 실제 QR 이미지로 인코딩/디코딩되어 통과(아직 채널은
메모리상 이미지 전달)". M1~M4의 전 스택(패킷·세션·ARQ·스트림·채팅)이 `Channel` 추상에만 의존하므로,
QR 코덱과 이미지 채널만 추가하면 상위 계층은 그대로 얹힌다. 마일스톤 작성 전 4개 설계 결정을
사용자와 확정했다(모두 반영):

1. **QR 생성 = segno**(순수 파이썬, Pillow 불필요).
2. **QR 디코드 = OpenCV** `cv2.QRCodeDetector`(Windows pip 설치 깔끔, numpy 배열 입력, M6 카메라 연동에도 사용).
3. **바이트 인코딩 = base64**(ASCII-safe — 디코더의 문자열 반환 시 바이너리 손상 방지; stdlib).
4. **범위 = 코덱 + ImageLoopbackChannel**(이미지 메모리 전달, 실제 QR 생성·디코드 통과). 실제 화면/카메라는 M6.

**의존성 도입**: 지금까지 stdlib-only였으나 본 마일스톤부터 `segno`, `opencv-python`, `numpy`가
필요하다(pyproject의 optional `[qr]`/`[optical]`에 segno/opencv 이미 선언; numpy는 opencv 의존). impl
시작 시 설치하고, QR 의존 테스트는 `pytest.importorskip`으로 라이브러리 부재 환경에서 skip되게 한다.

설계 원칙 유지: 코덱은 순수 함수(부작용 없음), 채널은 기존 `Channel` 인터페이스(`send_frame(bytes)`/
`recv_frame()->bytes|None`/`close()`)를 그대로 구현해 상위 스택 무수정.

## 공유 규약 (모든 태스크 준수)

- **이미지 포맷**: 2D `numpy.ndarray`, dtype `uint8`, 값은 0(검정)/255(흰색) 그레이스케일. QR 모듈을
  `scale`(기본 8) 픽셀로 확대, `border`(기본 4 모듈) quiet zone 포함. (디코더는 이 배열 또는 HxW/HxWx3 uint8 입력 허용.)
- **바이트 페이로드 표현**: `base64.b64encode(data)` → ASCII 문자열을 QR 텍스트로. 디코드 시 문자열 →
  `base64.b64decode` → 원본 바이트.
- **EC 레벨**: 코덱 함수는 `error='M'`(기본) 같은 EC 파라미터를 받는다(segno 표기 기준).

## 태스크 상세

### M5-T01: QR 인코더

- **변경 파일**: `photontcp/qr/encode.py`, `photontcp/qr/__init__.py`(re-export: `encode_frame`(.encode)와 `decode_frame`(.decode) 둘 다 — decode는 M5-T02가 생성하나 import는 테스트 시점에 둘 다 존재; `__all__` 명시)
- `import segno`, `import numpy as np`, `import base64`.
- `encode_frame(data: bytes, *, scale: int = 8, border: int = 4, error: str = "m") -> np.ndarray`:
  - `b64 = base64.b64encode(data).decode("ascii")`.
  - `qr = segno.make(b64, error=error)` (segno가 버전 자동 선택).
  - QR 모듈 매트릭스를 uint8 그레이스케일 배열로 렌더: 각 모듈을 `scale`×`scale` 픽셀로, 사방 `border` 모듈 quiet zone(흰색 255), 어두운 모듈=0, 밝은=255. (segno `qr.matrix` 순회로 직접 배열 구성하거나 `qr.save`를 PNG 버퍼→`cv2.imdecode`로 읽어 그레이스케일 변환 — 직접 매트릭스 구성 권장(의존성 최소·결정적).)
  - 반환 배열은 2D uint8.
- 모듈/함수 docstring + 공유 규약 명시.

### M5-T02: QR 디코더

- **변경 파일**: `photontcp/qr/decode.py`
- `import cv2`, `import numpy as np`, `import base64`.
- `decode_frame(image: np.ndarray) -> bytes | None`:
  - 입력이 컬러(HxWx3)면 그레이스케일 변환. `QRCodeDetector().detectAndDecode(image)`로 문자열 추출.
  - 검출 실패(빈 문자열/예외)면 `None`.
  - 추출 문자열을 `base64.b64decode`→bytes 반환. base64 디코드 실패도 `None`(손상 프레임 취급).
- (선택) `decode_frame`이 detector 인스턴스를 재사용하도록 모듈 전역/인자 허용(성능). 필수 아님.
- 모듈/함수 docstring.

### M5-T03: `ImageLoopbackChannel` (deps: M5-T01, M5-T02)

- **변경 파일**: `photontcp/channel/image_loopback.py`, `photontcp/channel/__init__.py`(re-export에 `ImageLoopbackChannel` 추가 — 기존 `Channel`/`LoopbackChannel` 유지)
- 기존 `LoopbackChannel`과 동일한 전이중 큐 구조이되, **큐에 raw bytes 대신 QR 이미지(numpy 배열)를 저장**한다:
  - `send_frame(self, frame: bytes)`: `encode_frame(frame, ...)`로 QR 이미지 생성 → outbox 큐에 push.
  - `recv_frame(self, timeout=None) -> bytes | None`: inbox에서 이미지 pop → `decode_frame(image)` → bytes 반환(디코드 실패면 손실로 간주, 다음 프레임 시도 또는 None). timeout 의미 보존.
  - `pair(cls, *, seed=None, loss=0.0, dup=0.0, scale=8, border=4, error="m", degrade=None) -> tuple[...]`: `LoopbackChannel.pair`와 유사. 노이즈는 **프레임(이미지) 단위 loss/dup**로(주입 `random.Random(seed)`), 선택적 `degrade`(이미지에 가우시안 노이즈/블러를 줘 QR EC 견딤 테스트 — 선택, 기본 None).
  - `close()` 멱등.
  - 이 채널은 `Channel` 인터페이스를 만족해 Session/ChatSession이 그대로 사용.
- docstring: 실제 QR 인코드/디코드가 경로에 들어감을 명시.

### M5-T04: QR 코덱 라운드트립 테스트 (deps: M5-T01, M5-T02)

- **변경 파일**: `tests/test_qr.py`
- 파일 상단 `pytest.importorskip("segno")`, `pytest.importorskip("cv2")`.
- `encode_frame`→`decode_frame` 라운드트립: (a) 짧은/긴 바이트열, (b) 모든 바이트값 포함(`bytes(range(256))`)으로 base64 바이너리 무결성, (c) 실제 `Packet.pack()` 바이트(헤더+payload)를 통과시켜 `Packet.unpack`이 CRC 검증 통과·필드 보존. (d) 빈 payload 패킷. 결정적(고정 입력).
- (선택) 손상 이미지/검출 실패 시 `decode_frame`이 `None` 반환.

### M5-T05: 이미지 채널 + 풀스택 통합 테스트 (deps: M5-T03)

- **변경 파일**: `tests/test_image_channel.py`
- `pytest.importorskip("segno")`/`importorskip("cv2")`.
- `ImageLoopbackChannel.pair()` + `ManualClock` 위에서 **기존 스택이 그대로 동작**함을 검증:
  - (a) 프레임 왕복: `Packet.pack()` 바이트 송신 → 반대편 `recv_frame()`이 동일 바이트 복원(QR 인코드/디코드 통과).
  - (b) 세션: 두 `Session`이 ImageLoopbackChannel 위에서 핸드셰이크 수립 → graceful 종료(양쪽 CLOSED).
  - (c) 신뢰 데이터/채팅: ESTABLISHED 후 바이트열(또는 ChatSession 메시지)을 보내 상대가 순서대로 무손실 수신(원본 일치). (성능 위해 payload는 작게.)
  - 유한 반복 상한 + 미수렴 시 pytest.fail. 결정적.

### M5-T06: QR 루프백 예제 (deps: M5-T03)

- **변경 파일**: `examples/qr_loopback.py`
- `ImageLoopbackChannel.pair()` 위에서 두 `Session`(또는 `ChatSession`)으로 세션 수립 → 메시지/바이트열 전송 → 상대가 수신·복원(MATCH) → 종료를 콘솔에 출력. "이 데이터는 실제 QR 이미지로 인코딩되어 전달됨"을 명시(예: 생성된 QR 이미지의 크기/모듈 수 출력). segno/cv2 미설치 시 안내 후 graceful exit. `__main__` 가드, 영어 출력.

## 완료 기준

1. `encode_frame(data)`→`decode_frame(image)` 라운드트립이 원본 바이트를 정확히 복원한다(짧은/긴/전 바이트값 포함, base64 바이너리 무결성).
2. 실제 `Packet.pack()` 바이트를 QR 이미지로 인코딩→디코딩하면 `Packet.unpack`이 CRC 검증을 통과하고 모든 필드가 보존된다.
3. `ImageLoopbackChannel`이 `Channel` 인터페이스를 만족하며, 송신 바이트가 QR 이미지를 거쳐 반대편에서 동일 바이트로 복원된다.
4. 기존 세션·ARQ·스트림·채팅 스택이 **수정 없이** `ImageLoopbackChannel` 위에서 동작한다 — 핸드셰이크 수립, graceful 종료, 신뢰 데이터/채팅 메시지의 순서 보장 무손실 전달.
5. 코덱은 순수(부작용 없음), 채널은 결정적(노이즈는 주입 seed)이며 QR 의존 테스트는 라이브러리 부재 시 skip된다.
6. `examples/qr_loopback.py`가 실제 QR 이미지를 거친 통신(원본 일치)과 종료를 출력한다.
7. `pytest`로 M1~M5 테스트가 전부 통과한다(M1~M4 회귀 없음; QR 의존 테스트는 설치 환경에서 통과, 미설치 시 skip).
