# M4 — 스트림 다중화 + 채팅 앱 (기본 통신 완성)

## 목표

하나의 PhotonTCP 세션 위에서 **여러 논리 스트림을 다중화**하고, 그 위에 **양방향 텍스트 채팅** 앱을
올린다. 각 스트림(stream_id≥1)은 독립적인 SR ARQ 인스턴스로 자기만의 신뢰성·순서·재전송을 가져
한 스트림의 손실이 다른 스트림을 막지 않는다(HOL 블로킹 제거). 채팅은 신뢰성 바이트 스트림 위에
길이접두(4B)+JSON 메시지 경계를 정의해 `{msg_id, timestamp, text}`를 주고받는다. 이 마일스톤이
끝나면 **손실 있는 LoopbackChannel + 가상 클럭에서 두 피어가 세션을 수립하고 양방향으로 텍스트
메시지를 무손실·순서대로 교환**하는 것이 결정적으로 검증된다 — 로드맵의 "기본 통신 완성".

## 배경

로드맵 3단계. M3까지 단일(사실상 stream 0/default) 신뢰성 데이터 경로가 있었고, 이제 이를 스트림
다중화로 일반화해 채팅을 올린다. 마일스톤 작성 전 4개 설계 결정을 사용자와 확정했다(모두 반영):

1. **다중화 = 스트림별 독립 ARQ.** `stream_id=0`은 제어(핸드셰이크/세션), `stream_id≥1`은 앱 스트림.
   각 앱 스트림은 자체 `ArqEndpoint`(독립 윈도우/시퀀스/재전송). 헤더의 `stream_id`(1바이트)와 `ArqEndpoint`의
   `stream_id` 인자를 그대로 활용. → **ACK 모호성도 해소**: 데이터 ACK/DATA/NACK는 stream_id≥1을 달고
   오므로 핸드셰이크 ACK(stream_id=0)와 라우팅만으로 구분된다(M3의 상태 기반 분기 단순화).
2. **메시지 포맷 = 길이접두 + JSON.** 4바이트 big-endian 길이 접두 + JSON(`{msg_id, timestamp, text}`)
   UTF-8 본문. 표준 라이브러리만(의존성 0), stdlib-only 기조 유지.
3. **스트림 수명 = 암묵적 경량.** 첫 DATA가 흐르면 스트림이 생기는 implicit open(명시 open 핸드셰이크
   없음). 스트림 종료는 세션 종료와 함께(별도 per-stream close 상태머신 없음).
4. **M3 리뷰 권장 4건은 M4 비범위(후속 연기)** — NACK 억제, 데이터 활동의 idle 타이머 갱신, 데이터
   재전송 상한/파라미터 노출, 종료 이벤트 정합성. 정리 마일스톤 또는 7단계 최적화로.

설계 원칙은 유지: **I/O 비의존 순수 컴포넌트 + 동기 펌프 + 주입 가상 클럭**(시각 `now` 주입, 결정적).
M2/M3 무손실·신뢰성 경로는 회귀 없이 유지한다.

**스트림 ID 규약**: `0`=제어(예약). `1`=기본 앱 스트림(양쪽 공유 — 레거시 `send()/recv()`와 채팅 기본).
`open_stream()`로 추가 스트림 할당 시 충돌 방지를 위해 **개시자=홀수(3,5,…), 응답자=짝수(2,4,…)** 패리티.

## 태스크 상세

### M4-T01: 스트림 다중화기 `StreamMux`

- **변경 파일**: `photontcp/stream/mux.py`, `photontcp/stream/__init__.py`(re-export)
- M1 `Packet`/`PacketType`, M3 `ArqEndpoint`/`RtoEstimator` 사용. **I/O·클럭 비의존**(시각 `now` 주입).
- 상수: `CONTROL_STREAM_ID = 0`, `DEFAULT_STREAM_ID = 1`.
- `@dataclass MuxOutput`: `packets: list[Packet]`, `delivered: dict[int, list[bytes]]`(stream_id→인도 바이트).
- `class StreamMux`:
  - `__init__(self, *, session_id, is_initiator, window_size=32, max_payload=200, rto_factory=lambda: RtoEstimator())`. 내부 `_endpoints: dict[int, ArqEndpoint]`, 다음 할당 id 카운터(패리티).
  - `_endpoint(self, stream_id) -> ArqEndpoint`: 없으면 lazy 생성(implicit open) — `ArqEndpoint(session_id=현재 세션id, send_isn=0, recv_isn=0, window_size, rto=rto_factory(), max_payload, stream_id=stream_id)`.
  - `open_stream(self) -> int`: 패리티 규약으로 새 stream_id 할당(개시자 홀수 3,5,…/응답자 짝수 2,4,…; 1은 기본 공유) 후 endpoint 생성, id 반환.
  - `send(self, stream_id, data, now) -> MuxOutput`: 해당 endpoint `send` 산출 패킷 반환.
  - `on_packet(self, pkt, now) -> MuxOutput`: `pkt.stream_id` endpoint로 라우팅(`on_packet`), 산출 패킷 + delivered(해당 stream_id 키로) 반환. stream_id=0이 들어오면 무시(제어는 Session이 처리) — 방어적.
  - `on_tick(self, now) -> MuxOutput`: 모든 endpoint `on_tick` 산출 패킷 합산.
  - `set_session_id(self, sid)`: 모든(및 향후) endpoint의 session_id 정렬(응답자가 핸드셰이크로 id 채택 시). M3의 `_sync_arq_session_id` 일반화.
- docstring + 스트림 ID 규약 주석.

### M4-T02: 채팅 메시지 코덱·재조립기

- **변경 파일**: `photontcp/app/codec.py`, `photontcp/app/__init__.py`(없으면 import 없는 placeholder; 최종 re-export는 T04)
- 순수·stdlib(`json`, `struct`)만. `dataclasses`.
- `@dataclass ChatMessage`: `msg_id: int`, `timestamp: float`, `text: str`.
- `encode_message(msg: ChatMessage) -> bytes`: JSON(utf-8) 직렬화 후 **4바이트 big-endian 길이 접두** + 본문.
- `class StreamReassembler`: 신뢰성 스트림에서 받은 바이트를 누적해 완전한 길이접두 프레임만 추출.
  - `feed(self, data: bytes) -> list[ChatMessage]`: 내부 버퍼에 append, 길이접두로 완전한 프레임을 가능한 만큼 잘라 JSON 파싱→`ChatMessage` 리스트 반환(불완전분은 버퍼 잔류). 메시지가 여러 청크에 걸치거나 한 청크에 여러 메시지여도 정확히 복원.
  - 손상/비정상 길이 방어(예: 비현실적 길이면 예외 또는 스킵 — 단순히 합리적 상한 두기).
- docstring.

### M4-T03: `StreamMux`를 Session에 통합 (deps: M4-T01)

- **변경 파일**: `photontcp/session/session.py`(M3)
- 단일 `ArqEndpoint`를 `StreamMux`로 교체. 내부 수신버퍼를 stream_id별로(`dict[int, list[bytes]]`).
- **레거시 호환 API 유지**(M3 테스트 회귀 방지): `send(self, data)` → `DEFAULT_STREAM_ID`로 송신, `recv(self) -> list[bytes]` → `DEFAULT_STREAM_ID` 인도분 반환.
- **스트림 인지 API 추가**: `open_stream() -> int`, `send_on(self, stream_id, data) -> None`, `recv_on(self, stream_id) -> list[bytes]`, `recv_all() -> dict[int, list[bytes]]`. 데이터 송신은 ESTABLISHED 한정(아니면 RuntimeError).
- **pump 라우팅(단순화)**: `pkt.stream_id == CONTROL_STREAM_ID(0)`이고 타입이 제어(SYN/SYN_ACK/FIN/FIN_ACK/HEARTBEAT)면 상태머신; `ACK`이고 stream_id=0이면 상태가 SYN_RCVD일 때 상태머신(핸드셰이크) 그 외 무시(제어 stream의 데이터 ACK는 없음); `stream_id ≥ 1`의 DATA/ACK/NACK → `StreamMux`. delivered는 stream_id별 수신버퍼에 누적.
  - 매 pump에서 상태머신 `on_tick` + mux `on_tick` 평가, 산출 패킷 송신.
- 응답자 session_id 채택 시 `mux.set_session_id(self._machine.session_id)` 정렬(M3 `_sync_arq_session_id` 일반화).
- M2/M3 무손실·신뢰성 경로 회귀 없이 유지. docstring 갱신.

### M4-T04: 채팅 앱 `ChatSession` (deps: M4-T02, M4-T03)

- **변경 파일**: `photontcp/app/chat.py`, `photontcp/app/__init__.py`(re-export)
- `ChatSession`: `Session`(M4-T03) + `StreamReassembler`(T02)를 묶는 채팅 엔드포인트.
  - `__init__(self, session: Session, clock, *, stream_id: int = DEFAULT_STREAM_ID)`. 내부 메시지 id 카운터, 수신 재조립기, 수신 메시지 큐.
  - `send_message(self, text: str) -> int`: `ChatMessage(msg_id=다음 id, timestamp=clock.now(), text)` 생성→`encode_message`→`session.send_on(stream_id, bytes)`. msg_id 반환.
  - `pump(self) -> list[ChatMessage]`: `session.pump()` 호출 후 `session.recv_on(stream_id)` 바이트를 재조립기에 feed해 새 `ChatMessage`들을 반환(+내부 큐 적재). (세션 수립/종료 진행도 이 pump가 겸함.)
  - 편의: `is_established`/`is_closed`/`connect`/`close` 위임, `received` 누적 접근.
  - timestamp는 주입 clock으로 결정적(실시간 금지).
- `app/__init__.py`: `ChatMessage`, `encode_message`, `StreamReassembler`, `ChatSession` re-export.
- docstring.

## 완료 기준

1. 한 세션 위에서 stream_id≥1의 여러 스트림이 **독립적으로** 신뢰성·순서 보장 전달을 하며, 한 스트림의 손실/구멍이 다른 스트림 인도를 막지 않는다(HOL 블로킹 없음).
2. 미지정 stream_id로 DATA가 오면 수신측이 implicit open으로 스트림을 만들어 인도하고, `open_stream`은 개시자/응답자 패리티로 충돌 없이 id를 할당한다.
3. 채팅 메시지는 길이접두+JSON으로 인코딩되어, 여러 청크로 쪼개지거나 한 청크에 여러 개가 와도 `StreamReassembler`가 정확히 복원한다(유니코드 text·msg_id·timestamp 보존).
4. 손실 `LoopbackChannel` + 가상 클럭에서 두 `ChatSession`이 세션 수립 후 **양방향 텍스트 메시지를 순서대로 무손실 교환**한다(손실은 ARQ 재전송으로 복구).
5. 레거시 `Session.send()/recv()`(기본 스트림) 동작이 유지되어 M2·M3 테스트가 회귀 없이 통과한다.
6. 모든 컴포넌트는 채널·클럭 비의존(시각 `now`/주입 clock)이며 결정적이다.
7. `examples/chat_loopback.py`가 손실 채널에서 양방향 채팅(전 메시지 순서대로 도착) → 종료를 출력한다.
8. `pytest`로 M1~M4 테스트가 전부 통과한다(M1~M3 회귀 없음).
