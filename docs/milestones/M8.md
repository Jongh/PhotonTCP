# M8 — 실물 광학 채널 (화면 디스플레이 + 카메라 캡처)

## 목표

QR 이미지를 **메모리 큐가 아니라 실제 화면에 띄우고 실제 카메라로 캡처**해 디코드하는
`OpticalChannel`을 추가해, "빛만으로 통신"을 실물 하드웨어 경로로 증명한다. 이 마일스톤이 끝나면
`ImageLoopbackChannel`이 시뮬레이션했던 QR 광학 경로를 화면+웹캠으로 그대로 대체할 수 있고,
세션·신뢰성·앱 계층은 한 줄도 바꾸지 않은 채(=`Channel` 인터페이스만 의존) 실제 광 링크 위에서 동작한다.
디바이스를 **메모리 페이크로 추상화**하므로 채널 로직(표시·캡처 스레드·dedup·종료)은 하드웨어 없이
결정적으로 테스트되고, cv2 실물 디바이스는 카메라/화면이 있을 때만 동작한다. 여기까지가 **v1.0 후보**의
기술적 전제다.

## 배경

M7-review "다음 단계"가 명시한 6단계: *"`ImageLoopbackChannel`을 실물로 교체. M7에서 깐 스레드
안전성(락·스레드로컬 detector) 토대 위에서 카메라 스레드를 도입하고, 권장 3(동시성 정확성)을 본격
검증. 여기까지 가면 '빛만으로 실 통신' 증명 → v1.0 후보."* 출처·전제:

- **M7-review 권장 3**: 스레드 안전성은 현재 "스모크" 수준 — *"M6 실물 광학에서 카메라 백그라운드
  스레드를 도입할 때 실제 동시 송수신 정확성(공유 rng 락 범위·스레드로컬 detector)을 본격 재검증"*. 본
  마일스톤이 그 카메라 스레드를 도입하는 지점이므로 동시성 정확성 검증을 포함한다.
- **기존 자산 재사용**: `photontcp/qr/encode.py`(`encode_frame`·`QRCapacityError`)·
  `photontcp/qr/decode.py`(`decode_frame`, 스레드로컬 detector)·`Channel` 추상(`channel/base.py`)을
  그대로 쓴다. `ImageLoopbackChannel`(`channel/image_loopback.py`)이 send=encode/recv=decode 구조의
  레퍼런스다 — 그 구조를 실물 디바이스 경계로 끌어올린다.
- **설계 원칙 유지**: 순수 코덱(encode/decode) + `Channel` 인터페이스 불변. 단, 광학은 본질적으로 I/O·
  스레드·타이밍을 동반하므로(이미 `ImageLoopbackChannel`도 큐·스레드 전제) 채널 구현 내부에 한해
  백그라운드 캡처 스레드를 둔다. **하드웨어 의존부는 전부 디바이스 추상 뒤로** 격리해 코어 로직은
  결정적으로 유지한다.

### 핵심 설계 결정 — 재캡처 dedup (채널 프레이밍 nonce)

카메라는 디스플레이 갱신보다 빠르게 돌아 **같은 QR을 여러 번 캡처**한다. 또 ARQ는 동일 패킷을
정당하게 반복 전송한다. 둘을 구분하려고 타이밍에 의존하지 않기 위해, 채널이 표시 직전에 **1바이트
롤링 카운터(nonce, mod 256)** 를 패킷 앞에 붙여(`channel_frame = nonce || packet`) QR로 인코딩한다.
수신 캡처 스레드는 디코드한 `channel_frame`을 **직전 인도분과 정확히 바이트 비교**해, 같으면(=같은
화면을 재캡처) 버리고, 다르면(=nonce가 증가한 새 프레임) nonce를 떼고 패킷만 인박스에 인도한다.
연속된 동일 ARQ 패킷도 nonce가 달라 각각 1회 인도된다. (대안: 프레임 사이 blank gap 삽입 — 타이밍
의존이라 채택하지 않음. nonce 방식은 결정적이라 페이크로 테스트 가능.)

### M8-T01: `optical` 서브패키지 + 디바이스 추상 + 메모리 페이크

- 새 서브패키지 `photontcp/optical/`을 만든다(채널과 분리 — 디바이스 추상·실물·채널을 한데 모음).
- `devices.py`에 두 추상 인터페이스(`abc.ABC`):
  - `DisplaySink`: `show(self, image: np.ndarray) -> None`(한 프레임을 표시) · `close(self) -> None`(멱등).
  - `CameraSource`: `read(self, timeout: float | None = None) -> np.ndarray | None`(가장 최근 캡처 프레임 1장,
    없으면 `None`) · `close(self) -> None`(멱등).
- 같은 파일에 **메모리 페이크 페어** — 하드웨어 없이 `OpticalChannel`을 결정적으로 테스트하기 위한
  인메모리 디바이스:
  - `MemoryDisplay(DisplaySink)`: `show`가 전달받은 이미지를 공유 `queue.Queue`(또는 "최근 1장" 슬롯)에
    넣는다.
  - `MemoryCamera(CameraSource)`: `read`가 그 큐에서 프레임을 꺼낸다. 카메라의 "같은 화면 반복 캡처"를
    재현하기 위해 **꺼낸 마지막 프레임을 유지**하다가 새 프레임이 오기 전까지 같은 프레임을 반복
    반환하는 모드를 지원한다(`repeat_last=True` 기본) — nonce dedup 경로를 실제로 자극.
  - `MemoryDisplay.pair_camera() -> tuple[MemoryDisplay, MemoryCamera]`(또는 모듈 함수
    `memory_device_pair()`)로 한 디스플레이와 그에 연결된 카메라를 만든다.
- 메모리 페이크는 손실/중복 노이즈를 흉내 낼 필요 없음(노이즈는 채널/ARQ 레벨에서 이미 검증됨) —
  목적은 채널의 표시/캡처/dedup/종료 **로직** 검증이다.
- `np` 외 새 서드파티 의존 없음. import 시 cv2·하드웨어 불필요.

### M8-T02: `OpticalChannel(Channel)` (deps: M8-T01)

- **변경 파일**: `photontcp/optical/channel.py`(추가)
- `OpticalChannel(Channel)` — 생성자는 `DisplaySink`와 `CameraSource`(주입)를 받는다. 직접 cv2를 알지
  못한다(디바이스 추상에만 의존 → 메모리/실물 모두 동일 코드로 구동).
  - 생성자 인자(권장): `display: DisplaySink`, `camera: CameraSource`, `*, scale/border/error`(encode 파라미터
    패스스루), `poll_interval`/`hold`(캡처 폴링·표시 유지 타이밍, 기본값 제공), `detector` 재사용 옵션.
  - 편의 팩토리 `OpticalChannel.pair(*, seed=None, **opts)` — **메모리 페이크 두 쌍을 교차 배선**해
    풀듀플렉스 인메모리 광학 페어 `(a, b)`를 만든다(테스트·예제용; `ImageLoopbackChannel.pair`와 동일한
    사용 감각). 실물은 `Cv2*` 디바이스를 직접 주입해 생성.
- `send_frame(frame: bytes)`: `nonce(1바이트, mod 256 롤링) || frame`을 만들어 `encode_frame`으로 QR
  이미지를 만들고 `display.show(image)` 호출. nonce는 인스턴스별 송신 카운터. 닫힌 채널은 무시.
  `QRCapacityError`는 그대로 전파(상위가 패킷 크기를 줄이도록 — M7과 일관).
- **백그라운드 캡처 스레드**: 생성 시(또는 첫 `recv_frame`/명시 `start()`) `threading.Thread`를 띄워
  루프에서 `camera.read(timeout=poll_interval)` → `decode_frame`(스레드로컬 detector) → 디코드 성공 시
  `channel_frame` 바이트를 **직전 인도분과 비교**: 같으면 재캡처로 간주해 버림, 다르면 nonce를 떼고
  패킷을 `inbox` 큐에 put 하고 "직전 인도분" 갱신. 디코드 실패(`None`)는 무시(손실 = ARQ가 재전송).
  스레드는 `_closed` 플래그로 깔끔히 종료.
- `recv_frame(timeout)`: `inbox` 큐에서 디코드된 패킷 바이트를 꺼낸다(없으면 `None`). 채널 자체는 항상
  whole-frame 반환(`Channel` 계약 유지).
- `close()`: 멱등 — `_closed` 설정 → 캡처 스레드 join → `display.close()`/`camera.close()` 호출.
- M7에서 깐 스레드 안전성에 의존: 캡처 스레드의 `decode_frame`은 스레드로컬 detector를 쓰고, 페이크
  RNG가 관여하면 락으로 보호됨 — 새 비보호 공유 상태를 만들지 말 것(인박스 큐·`_closed`만이 스레드
  경계 공유, 둘 다 thread-safe/원자적).

### M8-T03: cv2 실물 디바이스 (deps: M8-T01)

- **변경 파일**: `photontcp/optical/cv2_devices.py`(추가)
- `Cv2Display(DisplaySink)`: `show`가 `cv2.imshow(window, image)` + `cv2.waitKey(1)`로 한 프레임을 창에
  렌더. 창 이름·전체화면 옵션 인자. `close`는 `cv2.destroyWindow`.
- `Cv2Camera(CameraSource)`: 생성 시 `cv2.VideoCapture(index)` 오픈. `read`는 `cap.read()`로 한 프레임을
  잡아(BGR `ndarray`) 반환(`decode_frame`이 3채널을 grayscale로 변환 처리하므로 그대로 전달 가능).
  실패 시 `None`. `close`는 `cap.release()`.
- **하드웨어/창 없이도 import 가능**해야 한다 — `import cv2`만 모듈 상단, 실제 장치 오픈은 생성자에서.
  테스트는 `importorskip("cv2")`로 import 가능성만 확인(실제 카메라 동작은 수동/예제에서).
- 코덱·채널 로직 변경 없음(이 태스크는 순수 디바이스 어댑터).

### M8-T04: 동시성 정확성 검증 (deps: M8-T02)

- **변경 파일**: `tests/test_optical_concurrency.py`(추가)
- M7 권장 3을 본격 검증: `OpticalChannel.pair`(메모리 페이크)로 **양방향 동시 송수신**을 여러 스레드에서
  돌려, (1) 캡처 스레드가 `decode_frame`(스레드로컬 detector)을 동시 호출해도 크래시·교차오염 없음,
  (2) 페이크 디바이스/공유 상태에 데이터 레이스 없음(보낸 패킷 집합 == 받은 패킷 집합, 누락/중복 없음 —
  nonce dedup이 재캡처만 거르고 정당한 패킷은 모두 통과), (3) `close()`가 진행 중 캡처 스레드를 교착
  없이 종료. 가능하면 다수 패킷·다수 반복으로 통계적 신뢰성 확보(시드/순서 고정으로 재현 가능하게).
- 이건 "스모크"를 넘는 **정확성** 검증(보낸=받은 집합 동일)이 목표 — 단, 실시간 타이밍 민감 테스트는
  불안정하므로 페이크 디바이스의 결정적 핸드오프(큐 기반)에 기대 플레이키를 피한다.

### M8-T05: 광학 채널 테스트 (deps: M8-T02)

- **변경 파일**: `tests/test_optical_channel.py`(추가)
- 메모리 페어로 결정적 검증:
  - **왕복**: `OpticalChannel.pair`로 a→b, b→a 프레임이 정확히 복원된다.
  - **재캡처 dedup**: 카메라가 같은 화면을 N번 반복 반환해도(`repeat_last`) `recv_frame`은 패킷을 **1회만**
    인도한다.
  - **연속 동일 패킷**: 같은 바이트열 패킷을 연달아 두 번 보내면 nonce 덕에 **두 번 다** 인도된다(ARQ
    재전송이 묻히지 않음).
  - **종료**: `close()` 후 `send_frame`은 무시, `recv_frame`은 `None`, 캡처 스레드 join 완료(멱등 close).
  - **세션 통합**: 기존 `Session`/믹스를 `OpticalChannel.pair` 위에 올려 무손실 핸드셰이크+데이터 왕복이
    성립(채널 교체만으로 상위 계층 무변경 입증) — 짧은 메시지 1~2개 수준.
  - QR/numpy 부재 시 `importorskip`로 skip.

2. **광학 채널 왕복**: `OpticalChannel.pair` 위에서 a→b·b→a 프레임이 바이트 정확히 복원된다.
3. **재캡처 dedup**: 카메라가 같은 화면을 여러 번 반환해도 한 표시 프레임은 정확히 1회만 인도된다.
4. **연속 동일 패킷 보존**: 동일 바이트 패킷을 연달아 보내면 nonce 덕에 각각 인도되어 ARQ 재전송이
   묻히지 않는다.
5. **세션 무변경 구동**: 상위(`Session`/믹스/앱)를 한 줄도 바꾸지 않고 `OpticalChannel`로 핸드셰이크+데이터
   왕복이 성립한다(채널 인터페이스만 의존함을 입증).
6. **동시성 정확성**: 양방향 동시 송수신에서 보낸 패킷 집합 == 받은 패킷 집합(누락/오염 없음)이고,
   캡처 스레드의 동시 `decode_frame`(스레드로컬 detector)이 크래시 없이 동작한다(M7 권장 3 해소 — 스모크
   초과 정확성).
7. **종료 정합성**: `close()`가 멱등이며 캡처 스레드를 교착 없이 종료하고, 이후 send 무시·recv `None`.
8. **실물 디바이스 가용**: `Cv2Display`/`Cv2Camera`가 하드웨어·창 없이 import되고, `examples/optical_link.py`
   가 인메모리 모드로는 즉시, `--real`로는 웹캠/화면에서 동작한다(수동).
9. 모든 변경이 코덱 순수성·`Channel` 계약을 유지하고, **`pytest`로 M1~M8 테스트가 전부 통과**한다(M1~M7
   회귀 0). 카메라/cv2 의존 테스트는 부재 시 `importorskip`로 skip 한다.
