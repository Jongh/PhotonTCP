# M1 — 스켈레톤 · LoopbackChannel · 패킷 코덱 · CRC

## 목표

PhotonTCP의 최하위 두 계층(채널 추상화 + 패킷 직렬화)이 동작 상태가 된다.
이 마일스톤이 끝나면 임의의 바이트열을 PhotonTCP 패킷으로 직렬화해 `LoopbackChannel`을 통해
반대편으로 보내고, 받은 쪽에서 역직렬화·CRC 검증 후 원본 바이트열을 복원할 수 있다(에코).
또한 `LoopbackChannel`은 손실·중복·순서뒤바뀜·지연·비트손상을 **시뮬레이션**할 수 있어,
이후 신뢰성 계층(M3)을 실물 광학 없이 검증할 토대가 마련된다.

## 배경

로드맵 0단계에 해당한다. 전체 설계(`README.md`·대화 맥락)에서 모든 상위 계층은 교체 가능한
`Channel` 인터페이스와 고정 포맷 패킷에만 의존하기로 했다. 따라서 가장 먼저 (1) 채널 인터페이스와
가상 구현, (2) 패킷 헤더 직렬화 + CRC32 무결성을 확정해야 이후 세션(M2)·신뢰성(M3) 작업이
재해석 없이 얹힌다. 신규 프로젝트이므로 패키지 스켈레톤도 이 마일스톤에서 함께 세운다.

## 태스크 상세

### M1-T01: 패키지 스켈레톤 + `Channel` 추상 인터페이스

- **변경 파일**: `photontcp/__init__.py`, `photontcp/channel/__init__.py`, `photontcp/channel/base.py`, `photontcp/packet/__init__.py`
- `photontcp` 패키지와 하위 패키지(`channel`, `packet`)를 `__init__.py`로 만든다.
- `channel/base.py`에 `Channel` 추상 인터페이스를 정의한다 — `abc.ABC` 또는 `typing.Protocol` 중 하나로:
  - `send_frame(self, frame: bytes) -> None` — 한 프레임(=한 패킷 직렬화 결과) 송신.
  - `recv_frame(self, timeout: float | None = None) -> bytes | None` — 한 프레임 수신, 시간 내 없으면 `None`.
  - `close(self) -> None` — 채널 정리.
- 메서드 docstring으로 의미·블로킹 동작을 명시. 구현은 두지 않는다(추상).

### M1-T02: 패킷 타입·플래그 정의

- **변경 파일**: `photontcp/packet/types.py`
- `PacketType(IntEnum)` 정의: `SYN`, `SYN_ACK`, `ACK`, `DATA`, `NACK`, `FIN`, `FIN_ACK`, `HEARTBEAT`.
- `Flags(IntFlag)` 정의: `SYN`, `ACK`, `FIN`, `NACK` 등 보조 비트 플래그(헤더 `flags` 1바이트와 매핑).
- `PROTOCOL_VERSION = 1` 상수.

### M1-T03: CRC32 유틸

- **변경 파일**: `photontcp/packet/crc.py`
- `crc32(data: bytes) -> int` — 표준 라이브러리 `zlib.crc32`를 감싸 32비트 부호없는 정수로 반환.
- `verify(data: bytes, expected: int) -> bool` 헬퍼 제공.

### M1-T04: 패킷 헤더 직렬화/역직렬화 (deps: M1-T02, M1-T03)

- **변경 파일**: `photontcp/packet/header.py`
- 고정 22바이트 헤더, big-endian. 필드 순서/크기: `version(1)`, `type(1)`, `flags(1)`, `session_id(2)`, `stream_id(1)`, `seq(4)`, `ack(4)`, `window(2)`, `payload_len(2)`, `crc32(4)`. `struct` 포맷 `">BBBHBIIHHI"`.
- `@dataclass Packet`: 헤더 필드 + `payload: bytes`.
- `pack(self) -> bytes`:
  - 먼저 `crc32` 자리를 0으로 둔 헤더 + payload를 직렬화하고, 그 바이트열의 CRC32를 계산해 `crc32` 필드에 채운 뒤 최종 바이트열을 반환.
  - `payload_len`은 `len(payload)`로 자동 설정.
- `unpack(cls, raw: bytes) -> Packet`:
  - 헤더 파싱 → payload 분리 → `crc32` 필드를 0으로 둔 재직렬화로 CRC 재계산해 검증.
  - CRC 불일치 시 `ChecksumError`(이 모듈에 정의) 발생. 길이 부족·`payload_len` 불일치도 검증.
- 엣지케이스: payload 빈 경우, `payload_len`이 실제 길이와 다른 경우(손상), 입력이 22바이트 미만인 경우.

### M1-T05: `LoopbackChannel` + 노이즈 시뮬레이션 (deps: M1-T01)

- **변경 파일**: `photontcp/channel/loopback.py`
- 두 피어를 메모리 큐로 잇는 양방향 가상 채널. `queue.Queue` 기반, 스레드 안전.
- 팩토리: `LoopbackChannel.pair(**noise) -> tuple[Channel, Channel]` — 한쪽의 `send_frame`이 다른 쪽의 `recv_frame`으로 도착하도록 큐 2개를 교차 연결(전이중).
- 노이즈 파라미터(기본값 = 무손실, 결정적 재현 위해 `seed` 인자 지원):
  - `loss`(프레임 폐기 확률), `dup`(중복 송신 확률), `reorder`(순서 뒤바뀜 확률), `corrupt`(비트 손상 확률 — payload 일부 바이트 플립), `latency`/`jitter`(지연).
  - 난수는 주입된 `random.Random(seed)` 인스턴스로만 사용(테스트 재현성). `Date.now`/전역 난수 미사용.
- `recv_frame(timeout)`은 큐에서 꺼내며 timeout 만료 시 `None`.

### M1-T06: 단위 테스트 (deps: M1-T04, M1-T05)

- **변경 파일**: `tests/test_packet.py`, `tests/test_loopback.py`, `tests/__init__.py`
- `test_packet.py`: pack→unpack 라운드트립으로 모든 필드·payload 보존 확인; 빈 payload; CRC 손상 시 `ChecksumError`; 길이 부족 입력 예외.
- `test_loopback.py`: 무손실 채널에서 프레임 송→수신 일치; `loss=1.0`이면 항상 손실; `dup`/`corrupt`/`reorder`가 seed 고정 시 결정적으로 재현됨.

### M1-T07: 에코 예제 (deps: M1-T04, M1-T05)

- **변경 파일**: `examples/echo_loopback.py`
- `LoopbackChannel.pair()`로 두 끝점을 만들고, 한쪽에서 문자열을 `DATA` 패킷으로 직렬화해 송신 → 반대쪽에서 수신·역직렬화·CRC검증 후 원본 출력. 무손실 설정에서 입력=출력임을 콘솔로 시연.

## 완료 기준

1. `Packet`을 만들어 `pack()` 후 `unpack()` 하면 모든 헤더 필드와 payload가 원본과 동일하게 복원된다.
2. 직렬화 바이트의 임의 1바이트를 변조한 뒤 `unpack()` 하면 `ChecksumError`가 발생한다.
3. 무손실 `LoopbackChannel.pair()`에서 송신한 프레임이 반대편 `recv_frame()`으로 그대로 도착한다.
4. 동일 `seed`로 생성한 노이즈 채널은 손실/중복/손상 패턴이 결정적으로 재현된다(테스트 통과).
5. `examples/echo_loopback.py` 실행 시 입력 문자열과 수신 복원 문자열이 일치 출력된다.
6. `pytest`로 `tests/`가 전부 통과한다.
