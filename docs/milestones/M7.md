# M7 — 신뢰성 정리 / 하드닝

## 목표

M1~M6 동안 의도적으로 이월해 온 리뷰 권장·미반영 후속을 한데 모아 처리해, 핵심 전송·앱 계층의
**견고성·캡슐화·입력 방어·스레드 안전성**을 끌어올린다. 기능 추가가 아니라 **하드닝**이므로 외부에서
보이는 동작(채팅·파일·QR·세션)은 그대로 유지되면서, 손실·악성 입력·동시성 가장자리에서 더 안전해진다.
이 마일스톤이 끝나면 회고(`docs/reports/retro.md`)가 부각한 누적 권장이 대부분 해소되고, 다음 단계
(실물 광학)로 갈 때 스레드 모델 전환의 토대가 마련된다.

## 배경

회고에서 "M3/M4 잔여 권장을 흩어 넣지 말고 **신뢰성 정리 마일스톤으로 묶어** 처리"하라고 제안했고,
사용자가 4개 항목군(신뢰성 견고성·API 견고성·입력 방어·스레드 안전성)을 **모두** 선택했다. 출처:

- **M3-review 권장**: NACK 스톰 억제 · 데이터 활동의 세션 idle 타이머 갱신 · 데이터 재전송 상한 · 종료 이벤트 정합성.
- **M4-review 권장**: `ChatSession.received` 가변 노출 · `RtoEstimator` 설정 약결합(공개 접근자/clone).
- **M5-review 권장**: 단일 QR 용량 가드 · cv2 detector 스레드 안전성.
- **M6-review 권장**: 파일/채팅 기본 stream_id 공유 · 송신 progress가 큐잉 측정 · (OFFER 필드 검증).
- **M1-review 권장**: 공유 RNG 스레드 안전성.

설계 원칙 유지: 순수 컴포넌트 + 동기 펌프 + 주입 클럭. **회귀 0**(M1~M6 테스트 전부 통과)이 절대 조건.
스레드 안전성 변경은 **단일 스레드 결정성을 깨지 않는 방식**(락/스레드로컬)으로 — 기존 seed 기반 테스트
패턴을 보존한다.

- M7-T06: 세션/믹스 통합 — 파라미터 노출 · 데이터 idle 갱신 · clone 사용 · 스트림 acked 노출 (`session/session.py`, `stream/mux.py`) (deps: M7-T01, M7-T02, M7-T03)

## 태스크 상세

### M7-T01: ARQ 하드닝 (`reliability/arq.py`)

- **NACK 억제(R1)**: 수신측이 한 구멍(누락 seq)에 대해 **NACK를 한 번만** 보낸다(중복 NACK로 인한 동일 패킷 반복 재전송 방지). 이미 NACK한 seq 집합을 기억하고, 그 seq가 인도되거나 rcv_base가 지나가면 집합에서 제거. (RTO 재전송은 그대로 백업.)
- **데이터 재전송 상한(R3)**: `__init__(... , max_retx: int = 8)` 추가. `on_tick` 재전송 시 `retx_count`가 `max_retx` 초과하면 그 전송을 실패로 표시 — `ArqOutput.events`에 실패 이벤트(예: `ArqEvent.SEND_FAILED`나 단순 플래그)와 읽기 속성 `is_failed`/`failed` 노출. (세션이 이를 surface.)
- **acked 바이트 노출(A5-기반)**: 누적 ACK로 윈도우에서 제거되는 패킷의 payload 길이를 합산하는 카운터 `acked_bytes`(읽기 속성) 추가. (송신 진행률용.)
- 기존 동작·시그니처 호환 유지(새 인자는 기본값). 기존 ARQ 테스트 회귀 없음 확인.

### M7-T02: `RtoEstimator` 공개 설정 + `clone()` (`reliability/rto.py`)

- 공개 읽기 접근자: `initial_rto`/`min_rto`/`max_rto`(현재 비공개 `_` 속성을 공개 프로퍼티 또는 공개 필드로). 
- `clone(self) -> RtoEstimator`: 동일 설정(initial/min/max)의 **새 추정기**(상태 초기화) 반환. (스트림마다 독립 추정기 생성을 깔끔하게 — M4 리뷰의 약결합 해소.)
- 기존 동작 호환. 기존 RTO 테스트 회귀 없음.

### M7-T03: 세션 상태머신 정합성 + 활동 훅 (`session/state_machine.py`, `session/states.py`)

- **종료 이벤트 정합성(R4)**: `on_tick`의 idle 타임아웃에서 — 상태가 **FIN_WAIT 또는 CLOSE_WAIT(능동/수동 종료 진행 중)면 `CLOSED`** 이벤트로 종료(우리가 이미 닫는 중이므로), 그 외 활성 상태(ESTABLISHED 등)에서만 `TIMED_OUT`. 즉 "종료 진행 중 타임아웃"은 graceful close 완료로 간주해 일관되게 CLOSED를 낸다.
- **데이터 활동 idle 갱신 훅(R2)**: `note_data_activity(self, now: float) -> None` 추가 — 활성 상태면 `last_recv = now`로 갱신(데이터 평면 수신이 세션을 살아있게 유지). 상태 전이·패킷 생성 없음.
- (확인) `__init__`의 `control_rto`/`max_control_retries`는 M3에서 이미 존재 — 변경 없음(세션이 노출만).
- 기존 M2 무손실·M3 손실 동작 회귀 없음. docstring/상태표 갱신.

### M7-T04: 루프백 채널 스레드 안전성 (`channel/loopback.py`, `channel/image_loopback.py`)

- 공유 `random.Random`에 대한 모든 접근을 **`threading.Lock`으로 보호**(rng.random()/randint 등). 단일 스레드에서는 락 비경쟁이라 호출 순서·결과가 동일 → **기존 seed 기반 결정성·테스트 패턴 보존**. 동시 스레드에서 RNG 손상/경쟁만 방지.
- (M1 리뷰의 공유 RNG 스레드 안전성 해소. 결정성은 단일 스레드 한정이라는 기존 전제는 유지.)
- 기존 loopback/image 테스트 회귀 없음.

### M7-T05: QR 스레드 안전성 + 용량 가드 (`qr/decode.py`, `qr/encode.py`, `qr/__init__.py`)

- **스레드로컬 detector(T2)**: `decode.py`의 모듈 전역 단일 `cv2.QRCodeDetector`를 **`threading.local`** 기반으로 바꿔 스레드마다 독립 인스턴스 사용(M6 이후 카메라 스레드 대비). 단일 스레드 동작은 동일. `detector=` 인자 재사용 옵션 유지.
- **용량 가드(V1)**: `encode.py`에서 segno `DataOverflowError`(단일 QR 용량 초과)를 잡아 **명확한 예외**로 변환 — `qr` 모듈에 `QRCapacityError(Exception)` 정의하고 `encode_frame`이 초과 시 이 예외(원본 크기·base64 크기 정보 포함)를 raise. `qr/__init__.py`에 `QRCapacityError` re-export.
- 기존 QR/이미지 테스트 회귀 없음.

### M7-T06: 세션/믹스 통합 (deps: M7-T01, M7-T02, M7-T03) (`session/session.py`, `stream/mux.py`)

- **파라미터 노출(A1)**: `Session.__init__`에 `control_rto`/`max_control_retries`를 받아 `SessionStateMachine`에 전달.
- **데이터 idle 갱신(R2)**: `pump` 라우팅에서 stream_id≥1의 DATA/ACK/NACK를 ARQ로 보낼 때 `self._machine.note_data_activity(now)`도 호출(데이터 트래픽이 세션 idle 타이머를 살림).
- **clone 사용(A3)**: M4-T03에서 `rto` 인스턴스의 비공개 속성을 읽어 factory를 만들던 부분을 `rto.clone()` 사용으로 교체(약결합 제거).
- **스트림 acked 노출(A5)**: `StreamMux`에 `acked_bytes(stream_id) -> int`(해당 endpoint의 `acked_bytes`), `Session.acked_bytes(stream_id) -> int` 추가.
- **재전송 실패 surface(R3)**: ARQ가 어느 스트림에서 재전송 상한 초과로 실패하면 `Session`이 이를 알 수 있는 경로(예: `pump`이 해당 상황을 이벤트/플래그로 노출하거나 `recv_*`/속성). 최소: `Session.data_failed_streams()` 또는 pump 이벤트. 구현 명확히.
- 기존 M2~M6 동작·레거시 API(`send`/`recv`/`send_on`/`recv_on`) 회귀 없음. docstring 갱신.

### M7-T07: 앱 하드닝 (deps: M7-T06) (`app/chat.py`, `app/file.py`)

- **`ChatSession.received` 복사본(A2)**: 내부 리스트 대신 복사본(또는 튜플) 반환.
- **파일 전용 기본 stream(A4)**: `app/file.py`의 `FILE_STREAM_ID`를 채팅 기본(`DEFAULT_STREAM_ID`=1)과 **다른 예약 상수**(예: 2)로 분리해, 기본값만으로 채팅과 파일이 충돌 없이 병행 가능하게. (인자 오버라이드 유지.)
- **OFFER 필드 검증(V2)**: `FileReceiver`가 OFFER의 `name`(str)/`size`(int≥0)/`sha256`(hex str) 타입·존재를 검증 — 누락/타입 오류면 REJECT 송신 + FAILED(또는 명확한 거부). 손상 OFFER에 안전.
- **acked 기반 송신 progress(A5)**: `FileSender.progress`를 `session.acked_bytes(stream_id) / total`로 — 실제 전달된(ACK된) 바이트 기준. (큐잉 즉시 1.0이 되던 문제 해소.)
- `app/__init__.py`는 export 심볼 변동 없으면 그대로(필요 시 갱신). 기존 chat/file 테스트·예제 회귀 없음(파일 stream 기본 변경은 양측 동일 기본이라 일관 — 확인).

### M7-T08: 하드닝 테스트 (deps: M7-T06, M7-T07) (`tests/test_hardening.py`)

- 각 하드닝 동작을 결정적으로 검증:
  - NACK 억제: 한 구멍에 대해 NACK가 1회만 생성(여러 후속 DATA에도 추가 NACK 없음).
  - 데이터 재전송 상când: 상대가 영영 ACK 안 하면 `max_retx` 초과 후 ARQ/세션이 실패를 노출.
  - 종료 이벤트 정합성: FIN_WAIT/CLOSE_WAIT에서 타임아웃 시 `CLOSED`(not TIMED_OUT) — 가상 클럭으로.
  - 데이터 활동 idle 갱신: ESTABLISHED에서 데이터만 오가고 하트비트 없이도 idle_timeout 안에 죽지 않음.
  - `RtoEstimator.clone()`: 설정 동일·상태 초기화.
  - 스레드 안전성 스모크: 멀티스레드에서 `decode_frame`(스레드로컬 detector)·loopback RNG가 크래시 없이 동작(결정 검증 아님, 안전성 스모크).
  - QR 용량 가드: 과대 데이터 `encode_frame` → `QRCapacityError`.
  - 파일 전용 stream: 기본값으로 채팅+파일을 같은 세션에서 병행해도 서로 간섭 없음(서로 다른 stream).
  - OFFER 검증: 손상 OFFER → REJECT/FAILED.
  - acked progress: 전송 중 송신 progress가 ACK 도착에 따라 증가(즉시 1.0 아님), 완료 시 1.0.
- `importorskip`로 QR/스레드 테스트가 라이브러리 부재 시 skip.

## 완료 기준

1. **NACK 억제**: 한 구멍에 대해 NACK가 1회만 생성되고, 손실 복구는 여전히 정상 동작한다.
2. **데이터 재전송 상한**: 상대가 ACK하지 않으면 `max_retx` 초과 후 실패가 노출되며(무한 재전송 안 함), 정상 경로는 영향 없다.
3. **종료 이벤트 정합성**: FIN_WAIT/CLOSE_WAIT에서 타임아웃 시 `CLOSED` 이벤트로 종료된다(`TIMED_OUT` 아님). ESTABLISHED 타임아웃은 여전히 `TIMED_OUT`.
4. **데이터 활동 idle 갱신**: ESTABLISHED에서 데이터만 주고받아도(하트비트 없이) 세션이 idle로 죽지 않는다.
5. **`RtoEstimator.clone()`**: 동일 설정·상태 초기화의 새 추정기를 반환하고, 세션이 비공개 속성 접근 없이 이를 사용한다.
6. **스레드 안전성**: loopback RNG는 락으로 보호되고 cv2 detector는 스레드로컬이며, 멀티스레드 스모크에서 크래시가 없다. 단일 스레드 결정성·기존 테스트는 보존된다.
7. **QR 용량 가드**: 단일 QR 용량 초과 데이터를 `encode_frame`에 주면 `QRCapacityError`가 발생한다(불명확한 `DataOverflowError` 전파 안 함).
8. **파일 전용 stream**: 기본값만으로 채팅과 파일 전송을 같은 세션에서 병행해도 서로 간섭하지 않는다.
9. **OFFER 검증**: 손상/누락 필드 OFFER는 REJECT/FAILED로 안전 처리된다.
10. **acked 기반 progress**: `FileSender.progress`가 ACK된 바이트 기준으로 증가하며 큐잉 즉시 1.0이 되지 않고, 완료 시 1.0이다.
11. 모든 변경은 순수/결정적 원칙을 유지하고, **`pytest`로 M1~M7 테스트가 전부 통과**한다(M1~M6 회귀 없음).
