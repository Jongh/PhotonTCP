# M3 — 신뢰성 계층 (Selective Repeat ARQ)

## 목표

PhotonTCP가 손실·중복·순서뒤바뀜이 있는 채널 위에서도 **신뢰성 있는 순서 보장 데이터 전송**을
제공한다. Selective Repeat(SR) ARQ로 송신 슬라이딩 윈도우·선택적 재전송을 하고, 수신측은 재정렬·중복
제거 버퍼로 누적 ACK + 선택 NACK를 생성한다. 적응형 RTO로 손실을 타임아웃 재전송하며, 시퀀스 번호는
32비트 wraparound-safe 산술로 비교한다. 또한 M2의 제어 패킷(SYN/SYN_ACK/FIN)도 손실 시 재전송되고
수립이 끝내 실패하면 타임아웃 처리된다. 이 마일스톤이 끝나면 **손실 시뮬레이션 LoopbackChannel +
가상 클럭에서 양 피어가 세션을 수립하고 임의 바이트열을 무손실·순서대로 주고받은 뒤 종료**하는 것이
결정적으로 검증된다.

## 배경

로드맵 2단계. M1(패킷/채널), M2(무손실 가정 세션) 위에 신뢰성을 얹는다. 마일스톤 작성 전 4개 설계
결정을 사용자와 확정했다(모두 반영):

1. **ARQ = Selective Repeat(SR)** — 손실 패킷만 선택 재전송, 수신 재정렬 버퍼 사용.
2. **ACK = 누적 ACK + 선택 NACK** — 헤더의 누적 `ack` 필드와 `NACK` 패킷 타입을 그대로 활용(SACK 유사).
3. **시퀀스 산술 = 32비트 wraparound-safe** — `seq`/`ack`가 32비트이므로 mod 2³² 시리얼 번호 비교(RFC1982 류). M1 리뷰 권장 ①을 여기서 해소.
4. **범위 = DATA + 제어 모두** — ARQ 타이머/재전송을 DATA뿐 아니라 M2 핸드셰이크/FIN에도 적용하고, 수립 타임아웃(재시도 한도 초과 시 실패)을 추가. M2 리뷰 권장 1을 해소.

설계 원칙은 M2와 동일하게 유지한다: **I/O 비의존 순수 컴포넌트 + 동기 펌프 + 주입 가상 클럭**(시각은
`now: float` 주입, 랜덤·실시간 미사용으로 결정적). 동시성은 단일 스레드(공유 RNG·스레드 안전성은
계속 비범위 — 단일 스레드 모델이라 무의미).

**의식적 비범위/이월**: (a) 패킷 간 FEC/파운틴 코드는 로드맵 7단계로 이월. (b) 스트림 다중화·채팅 앱은
M4. (c) `LoopbackChannel`의 전달 지연(latency/jitter)은 동기·클럭 비주입 큐 구조상 본 마일스톤에서
구현하지 않고, RTT/RTO 검증은 **가상 클럭을 송신~ACK 사이에 전진시키는 방식**으로 결정적으로 수행한다
(전달 지연 채널은 7단계 최적화로 이월).

## 태스크 상세

### M3-T01: 32비트 시리얼 번호 산술

- **변경 파일**: `photontcp/reliability/serial.py`, `photontcp/reliability/__init__.py`(없을 때만 import 없는 placeholder 생성; 최종 re-export는 T03이 덮어씀 — 이미 있으면 건드리지 말 것)
- 32비트 시리얼 번호(`SEQ_MOD = 2**32`) 비교/연산 함수:
  - `seq_add(a, n) -> int` = `(a + n) % SEQ_MOD`.
  - `seq_lt(a, b) -> bool`, `seq_leq(a, b)`, `seq_gt`, `seq_geq` — RFC1982 류 wraparound-safe 비교(`((a - b) % 2**32)`의 상/하반 판정). a==b 처리 명확히.
  - `seq_diff(a, b) -> int` = 부호 고려한 순환 거리(필요 시).
- 순수 함수, 표준 라이브러리만. docstring + 경계(랩 지점, a==b) 설명.

### M3-T02: 적응형 RTO/RTT 추정기

- **변경 파일**: `photontcp/reliability/rto.py`
- `class RtoEstimator`: Jacobson/Karels 방식. `__init__(self, *, initial_rto: float, min_rto: float, max_rto: float)`. 내부 SRTT/RTTVAR.
  - `on_sample(self, rtt: float) -> None`: 첫 샘플은 SRTT=rtt, RTTVAR=rtt/2; 이후 표준 갱신식(α=1/8, β=1/4). RTO = SRTT + 4·RTTVAR, [min,max]로 클램프.
  - `rto(self) -> float`: 현재 RTO.
  - `on_timeout(self) -> None`: 지수 백오프(RTO 2배, max 클램프) — 재전송 시 호출.
- 순수·결정적(시간은 호출자가 rtt로 전달). docstring.

### M3-T03: Selective Repeat ARQ 엔진 (deps: M3-T01, M3-T02)

- **변경 파일**: `photontcp/reliability/arq.py`, `photontcp/reliability/__init__.py`(공개 API re-export로 작성)
- M1 `Packet`/`PacketType`(DATA/ACK/NACK)/`Flags`, T01 serial, T02 RTO 사용. **I/O·클럭 비의존**(시각은 `now` 주입). 랜덤 미사용.
- 출력 컨테이너 `@dataclass ArqOutput`: `packets: list[Packet]`(보낼 것), `delivered: list[bytes]`(상위로 인도된 순서 데이터), `events: list`(선택).
- `class ArqEndpoint`(또는 SR sender+receiver 결합):
  - `__init__(self, *, session_id, send_isn, window_size, rto: RtoEstimator)`. 송신 next_seq=send_isn, 송신 윈도우(미ACK 패킷 맵: seq→(packet, send_time, retx_count)), 수신 rcv_base(기대 seq)·재정렬 버퍼.
  - `send(self, data: bytes, now) -> ArqOutput`: 윈도우 여유가 있으면 DATA 패킷(seq=next_seq) 생성·기록·송신, next_seq 전진. 윈도우 가득이면 큐잉(내부 보류 후 ACK로 윈도우 열릴 때 방출) 또는 송신 거부 정책 명확히. payload 분할(MTU/페이로드 상한)은 호출자가 청크로 주거나 엔진이 `max_payload`로 분할 — 분할 위치 명시.
  - `on_packet(self, pkt, now) -> ArqOutput`:
    - **DATA 수신**: seq가 rcv_base면 인도(delivered)하고 rcv_base 전진, 버퍼에 연속분 있으면 함께 인도. 미래 seq(윈도우 내)는 버퍼에 저장. 중복(이미 인도/버퍼)·rcv_base 미만은 폐기. 어느 경우든 누적 ACK(=다음 기대 seq) 생성. 버퍼에 구멍이 있으면 누락 seq에 대한 NACK 동봉(선택).
    - **ACK 수신**: 누적 ack 이하 미ACK 패킷을 윈도우에서 제거(윈도우 전진). Karn 규칙: 재전송 안 된 패킷의 ACK에 한해 RTT 샘플(now - send_time)을 `rto.on_sample`. flow control: 상대 `window` 필드를 송신 윈도우 상한에 반영.
    - **NACK 수신**: 지정 seq를 즉시 선택 재전송(send_time 갱신).
  - `on_tick(self, now) -> ArqOutput`: 미ACK 패킷 중 `now - send_time >= rto.rto()`인 것을 재전송(retx_count++, send_time=now), `rto.on_timeout()` 호출. 재전송 시 RTT 샘플 금지(Karn).
- 시퀀스 비교·윈도우 경계는 전부 T01 serial 함수로(랩 안전). docstring + 동작표 주석.

### M3-T04: 제어 패킷 재전송 + 수립 타임아웃 (deps: M3-T02)

- **변경 파일**: `photontcp/session/state_machine.py`(M2), `photontcp/session/states.py`(M2 — 이벤트 추가)
- M2 세션 상태머신을 확장해 **제어 패킷 손실에 견디게** 한다:
  - 미ACK 제어 패킷 기억: SYN(SYN_SENT), SYN_ACK(SYN_RCVD), FIN(FIN_WAIT/CLOSE_WAIT). 각 상태의 `on_tick`에서 `now - last_control_send >= rto` 경과 시 해당 제어 패킷 재전송(재시도 카운터++). RTO는 주입(`RtoEstimator` 또는 고정 `control_rto` 파라미터) — 결정성을 위해 주입형으로.
  - **수립/종료 타임아웃**: 재시도 카운터가 `max_retries` 초과하면 포기 → 새 이벤트 `SessionEvent.CONNECT_FAILED`(수립 단계) 또는 기존 `TIMED_OUT`(범용) 발생 + 상태 CLOSED. `states.py`에 `CONNECT_FAILED` 추가.
  - 정상 진행(SYN_ACK/ACK/FIN_ACK 수신) 시 해당 미ACK 제어 패킷·카운터 클리어.
- 기존 무손실 동작은 회귀 없이 유지(M2 테스트 통과). `__init__`에 재전송 관련 파라미터(예: `control_rto`, `max_retries`) 추가(기본값 둬 M2 호출 호환).

### M3-T05: ARQ 데이터 경로를 Session에 통합 (deps: M3-T03)

- **변경 파일**: `photontcp/session/session.py`(M2)
- `Session`에 신뢰성 데이터 경로를 추가한다(상태머신은 제어, ARQ는 데이터 담당):
  - 내부에 `ArqEndpoint` 보유. `send(self, data: bytes) -> None`: ESTABLISHED일 때만 허용, `arq.send(data, clock.now())` 산출 패킷 송신(아니면 예외 또는 보류 정책 명시).
  - `pump()` 라우팅 확장 — 수신 패킷을 타입/상태로 분기:
    - `SYN/SYN_ACK/FIN/FIN_ACK/HEARTBEAT` → 세션 상태머신.
    - `DATA/NACK` → ARQ.
    - `ACK` → **상태가 SYN_RCVD이면 세션 상태머신(핸드셰이크 완성), 그 외 ESTABLISHED 이후엔 ARQ(데이터 ACK)**. (핸드셰이크 ACK가 소비되는 유일 지점이 SYN_RCVD이므로 모호성 해소.)
  - `pump()`는 세션 이벤트와 함께 **ARQ로 인도된 데이터**를 전달해야 한다 — 반환형을 확장하거나(`(events, delivered)`) `Session.recv() -> list[bytes]`/수신 큐를 둔다. 인터페이스 선택을 명확히 하고 일관되게.
  - `arq.on_tick`/상태머신 `on_tick` 모두 매 pump에서 평가, 양쪽 산출 패킷 송신.
  - (M2 리뷰 권장 2) `Session.session_id` 등 읽기 접근자 추가(선택, 저위험).
- 무손실 경로 회귀 없이 유지.

### M3-T06: ARQ 엔진 테스트 (deps: M3-T03)

- **변경 파일**: `tests/test_arq.py`
- 손실/중복/순서뒤바뀜 `LoopbackChannel.pair(seed=...)` + `ManualClock`로 두 `ArqEndpoint`를 펌프 구동(또는 엔진을 직접 손으로 펌프). 검증:
  - 무손실: 보낸 데이터가 순서대로 전부 인도.
  - 손실: 일부 DATA 폐기 → 가상시간을 RTO 이상 전진 → 재전송으로 결국 전부 인도(손실 없음).
  - 순서뒤바뀜: 수신 재정렬로 순서대로 인도.
  - 중복: 중복 DATA가 한 번만 인도(dedup).
  - NACK: 구멍 발생 시 선택 재전송으로 복구.
  - RTT/RTO: 송신~ACK 사이 가상시간 전진으로 RTT 샘플 반영, RTO 결정적.
  - 윈도우: 윈도우 한도 준수(미ACK 수 ≤ window).

## 완료 기준

1. 손실·중복·순서뒤바뀜 `LoopbackChannel`(seed 고정) + `ManualClock`에서, `ArqEndpoint`가 보낸 바이트열을 상대가 **순서대로 무손실 인도**한다(손실분은 RTO 재전송·NACK로 복구, 중복은 1회만 인도).
2. 시퀀스 비교는 32비트 wraparound-safe 산술을 사용하며 랩 경계에서 올바르게 동작한다(단위 테스트 통과).
3. 적응형 RTO가 RTT 샘플로 갱신되고 min/max 클램프·타임아웃 지수 백오프가 동작한다.
4. 송신 윈도우가 한도를 지키고, 수신 광고 윈도우(`window`)를 송신측이 반영한다(기본 흐름 제어).
5. 손실 채널에서 SYN/SYN_ACK/FIN이 재전송되어 **핸드셰이크 수립과 graceful 종료가 완료**되며, 상대가 끝내 응답 없으면 재시도 한도 후 `CONNECT_FAILED`(또는 TIMED_OUT) + CLOSED로 수립이 타임아웃된다.
6. `Session`이 ESTABLISHED 후 `send(data)`한 바이트열을 손실 채널을 거쳐 상대가 순서대로 전부 수신한다(엔드투엔드 신뢰 전송).
7. 모든 컴포넌트는 채널·클럭 인스턴스에 비의존(시각 `now` 주입)하며 동일 입력에 결정적이다.
8. `examples/reliable_loopback.py`가 손실 채널에서 신뢰 전송(원본 일치) → 종료 흐름을 출력한다.
9. `pytest`로 M1·M2·M3 테스트가 전부 통과한다(M1·M2 회귀 없음).
