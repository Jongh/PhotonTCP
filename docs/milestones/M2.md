# M2 — 세션 계층 (핸드셰이크 · 종료 · 하트비트)

## 목표

PhotonTCP에 연결 지향 세션 계층이 생긴다. 두 피어가 3-way 핸드셰이크(SYN/SYN_ACK/ACK)로
세션을 수립하고, `session_id`와 초기 시퀀스 번호(ISN)를 합의하며, graceful 종료(FIN/FIN_ACK)로
양쪽이 깔끔히 닫히고, 유휴 중에는 HEARTBEAT를 주고받아 링크 생존을 확인한다. 세션 로직은
I/O와 완전히 분리된 **순수 상태머신**으로 구현되어, 무손실 `LoopbackChannel`과 **주입 가능한
가상 클럭** 위에서 핸드셰이크·종료·하트비트·타임아웃이 real sleep 없이 결정적으로 검증된다.

## 배경

로드맵 1단계. M1에서 패킷 직렬화와 교체 가능한 채널이 완성됐고, 이제 그 위에 세션 관리가 필요하다.
구현 전 4개 설계 결정을 사용자와 확정했다(모두 본 마일스톤에 반영):

1. **실행/동시성 모델 = 순수 상태머신 + 동기 펌프.** 세션 로직은 입력(도착 패킷·타이머 틱) →
   출력(보낼 패킷 목록 + 액션)인 순수 함수형 상태머신. 백그라운드 스레드 없이 명시적
   `pump()`/`step()` 으로 구동 → 단일 스레드 결정적 테스트.
2. **시간 소스 = 주입 가능한 가상 클럭.** `Clock` 인터페이스를 주입해 테스트는 가상 시간을 임의
   전진시키고, 실운영은 monotonic 클럭을 주입.
3. **M2 범위 = 무손실 가정.** 제어 패킷 손실 대응(타임아웃 재전송)은 **M3(ARQ)로 연기**. 단
   상태머신은 향후 타이머/재전송 훅을 끼울 수 있도록 타이머 이벤트 입력을 받는 형태로 설계.
4. **M1 리뷰 권장 이슈는 M3로 연기**(본 마일스톤 비범위): ① `seq`/`ack` 범위 정책(mod 2³²)은
   시퀀스 산술의 주인인 M3에서 확정, ② 공유 RNG·스레드 안전성은 단일 스레드 모델이라 M2에선 무의미.

상태 전이: `CLOSED → SYN_SENT/SYN_RCVD → ESTABLISHED → FIN_WAIT/CLOSE_WAIT → CLOSED`.

## 태스크 상세

### M2-T01: 클럭 추상화

- **변경 파일**: `photontcp/session/clock.py`, `photontcp/session/__init__.py`(없으면 빈 패키지 초기화 생성 — 단, 재노출은 T04가 담당하므로 여기서는 import 없는 placeholder만; T04가 덮어쓴다)
- `Clock` 인터페이스(`typing.Protocol` 또는 `abc.ABC`): `now(self) -> float` (단조 증가 초 단위).
- `ManualClock(Clock)`: 테스트용. `now()` 는 내부 시각 반환, `advance(self, dt: float) -> None` 로 가상 시간 전진, `set(self, t: float)` 선택. 초기값 0.0.
- `MonotonicClock(Clock)`: 실운영용. `time.monotonic()` 위임.
- 전역 시간/`Date`/`time.time` wall-clock 사용 금지(주입만). 모듈/클래스 docstring 포함.
- 주의: `__init__.py` 충돌 회피 — T04가 최종 re-export 버전을 작성하므로, 이 태스크는 `__init__.py`가 없을 때만 최소 placeholder를 만들고 있으면 건드리지 않는다.

### M2-T02: 세션 상태·이벤트·액션 정의

- **변경 파일**: `photontcp/session/states.py`
- `class SessionState(Enum)`: `CLOSED`, `SYN_SENT`, `SYN_RCVD`, `ESTABLISHED`, `FIN_WAIT`, `CLOSE_WAIT`.
- 상태머신 출력 액션을 표현할 경량 타입(예: `@dataclass` 또는 `Enum`): 최소한 상위 계층에 알릴 이벤트 —
  `ESTABLISHED`(연결됨), `PEER_CLOSED`(상대가 FIN), `CLOSED`(완전 종료), `TIMED_OUT`(하트비트 부재로 사망).
  형태는 구현자가 정하되 상태머신 반환값에서 소비 가능해야 한다.
- 상수 기본값: `DEFAULT_HEARTBEAT_INTERVAL`, `DEFAULT_IDLE_TIMEOUT`(>interval) 정의(초 단위, 예: 1.0 / 3.0).
- 모듈 docstring 포함.

### M2-T03: 순수 세션 상태머신 (deps: M2-T02)

- **변경 파일**: `photontcp/session/state_machine.py`
- M1의 `Packet`/`PacketType`/`Flags`(`photontcp.packet`)와 T02의 `SessionState`를 사용. **I/O·채널·클럭 인스턴스에 의존하지 않는다**(시각은 `now: float` 인자로 받음).
- `SessionStateMachine` 구성: `__init__(self, *, is_initiator: bool, session_id: int, isn: int, heartbeat_interval: float, idle_timeout: float)`.
  - 결정성을 위해 `session_id`·`isn`(초기 seq)은 **주입**받는다(랜덤·시간 사용 금지).
- 입력 API(각 호출은 보낼 패킷 목록 + 발생 이벤트 목록을 반환):
  - `connect(self, now) -> Output` : 개시자 전용. CLOSED→SYN_SENT, SYN 패킷 생성(seq=isn, 제안 session_id).
  - `on_packet(self, pkt: Packet, now) -> Output` : 도착 패킷 처리(상태 전이 + 응답 패킷 + 이벤트).
  - `on_tick(self, now) -> Output` : 타이머 평가(하트비트 송출, idle 타임아웃 감지).
  - `close(self, now) -> Output` : 로컬 종료 개시(FIN 송출, 상태 전이).
  - `Output`은 `(packets_to_send: list[Packet], events: list[...])` 형태(명명된 dataclass 권장).
- 핸드셰이크(무손실 가정):
  - 개시자: `connect`→SYN_SENT(SYN). `SYN_ACK` 수신→ESTABLISHED + ACK 송출 + `ESTABLISHED` 이벤트.
  - 응답자: `SYN` 수신(CLOSED)→SYN_RCVD + SYN_ACK 송출(ack=상대 isn+1, session_id 합의). `ACK` 수신→ESTABLISHED + `ESTABLISHED` 이벤트.
- 종료(graceful, 무손실) — **대칭 auto-close**(구현 확정, M2 impl 리뷰 반영):
  - 능동: `close`(ESTABLISHED)→FIN 송출, FIN_WAIT.
  - 수동: `FIN` 수신(ESTABLISHED)→**FIN_ACK + 자기 FIN을 함께 송출**, CLOSE_WAIT + `PEER_CLOSED` 이벤트(수동측이 별도로 `close()`를 호출할 필요 없음).
  - 각 측은 "자기 FIN이 ACK됨 **그리고** 상대 FIN을 ACK함" 두 조건이 모두 충족될 때 CLOSED + `CLOSED` 이벤트. 따라서 **단일 active close()로 양쪽이 CLOSED에 수렴**한다(동시 close도 동일). `close()`는 ESTABLISHED에서만 동작하고 그 외엔 멱등 no-op(중복 FIN 방지). 반이중(half-close)은 비지원 — 후속 고려.
- 하트비트/타임아웃(`on_tick` 기준):
  - ESTABLISHED에서 마지막 **송신** 이후 `heartbeat_interval` 경과 시 HEARTBEAT 송출(송신 시각 갱신).
  - 마지막 **수신**(임의 패킷/하트비트 포함) 이후 `idle_timeout` 경과 시 `TIMED_OUT` 이벤트 + 상태 CLOSED.
  - `last_send`/`last_recv` 시각은 전달된 `now`로만 갱신(가상 클럭 호환).
- `session_id` 검증: 합의된 세션과 다른 `session_id` 패킷은 무시(드롭). 잘못된 상태의 패킷도 안전하게 무시.
- 모듈/클래스 docstring + 상태표 주석.

### M2-T04: 세션 드라이버/동기 펌프 + 패키지 re-export (deps: M2-T01, M2-T03)

- **변경 파일**: `photontcp/session/session.py`, `photontcp/session/__init__.py`
- `Session`: 순수 상태머신(T03)을 `Channel`(M1)·`Clock`(T01)과 묶는 동기 드라이버. 백그라운드 스레드 없음.
  - `__init__(self, channel: Channel, clock: Clock, *, is_initiator: bool, session_id: int, isn: int, heartbeat_interval=DEFAULT_HEARTBEAT_INTERVAL, idle_timeout=DEFAULT_IDLE_TIMEOUT)`.
  - `connect(self) -> None`: 개시자 핸드셰이크 시작(상태머신 `connect` 호출 후 산출 패킷을 채널로 송신).
  - `close(self) -> None`: 종료 개시(상태머신 `close` 산출 패킷 송신).
  - `pump(self, max_frames: int | None = None) -> list[event]`: 한 사이클 — (1) 채널에서 가용 프레임을 비블로킹/짧은 timeout으로 드레인해 `Packet.unpack` 후 `on_packet`, (2) `on_tick(clock.now())` 평가, (3) 산출된 모든 패킷을 `pack()` 해 채널로 송신, (4) 발생 이벤트 반환. 손상/역직렬화 실패 프레임은 무시(M2 무손실이라 정상 미발생이나 방어적으로).
  - 편의: `run_until(self, predicate, max_iters)` 또는 테스트가 양 피어 `pump()`를 번갈아 호출해 진전시키는 패턴 지원. `is_established`, `is_closed`, `state` 속성 노출.
- `photontcp/session/__init__.py`: 공개 API 재노출 — `Session`, `SessionState`, `Clock`, `ManualClock`, `MonotonicClock`, `SessionStateMachine`(및 이벤트 타입). `__all__` 명시. (M1 리뷰의 패키지 재노출 일관성 규약을 따른다.)

### M2-T05: 핸드셰이크·종료 테스트 (deps: M2-T04)

- **변경 파일**: `tests/test_session_handshake.py`
- 무손실 `LoopbackChannel.pair()` + `ManualClock`로 개시자/응답자 `Session` 2개 구성.
- 검증: 양 피어를 번갈아 `pump()` 하면 둘 다 `ESTABLISHED` 도달(상태·`ESTABLISHED` 이벤트), `session_id`/ack 번호가 핸드셰이크 규칙대로 합의됨. 이후 한쪽 `close()` → 펌프 → **양쪽 모두 `CLOSED`** 도달(graceful). 수동측에서 `PEER_CLOSED` 이벤트 관측. 진전 루프는 유한 반복 상한으로 무한루프 방지.

### M2-T06: 하트비트·타임아웃 테스트 (deps: M2-T04)

- **변경 파일**: `tests/test_session_heartbeat.py`
- ESTABLISHED 상태에서 `ManualClock.advance(heartbeat_interval+ε)` 후 `pump()` 하면 HEARTBEAT가 송출되어 상대가 수신함을 확인(가상 시간, real sleep 없음).
- 한쪽 pump를 멈춰 하트비트를 끊고 상대 클럭만 `idle_timeout` 초과로 전진시킨 뒤 `pump()` → `TIMED_OUT` 이벤트 + 상태 CLOSED 확인.
- 정상적으로 하트비트가 계속 오가는 동안에는 타임아웃이 발생하지 않음을 확인(경계값). `ManualClock` 자체 동작(advance/now)도 가볍게 커버.

### M2-T07: 세션 수립→종료 데모 예제 (deps: M2-T04)

- **변경 파일**: `examples/session_loopback.py`
- 무손실 loopback + `ManualClock`로 두 `Session`을 만들어, 펌프 루프로 핸드셰이크 수립 로그 출력 → 가상 시간을 전진시켜 하트비트 1~2회 송출 시연 → 한쪽 `close()` 후 양쪽 CLOSED 도달까지 진행하고 상태 전이를 콘솔에 출력. 레포 루트에서 `python examples/session_loopback.py` 실행 가능(필요 시 `sys.path` 보정), `__main__` 가드.

## 완료 기준

1. 무손실 `LoopbackChannel` + `ManualClock`에서 개시자/응답자를 번갈아 `pump()` 하면 양 피어가 `ESTABLISHED`에 도달하고, `session_id`와 ACK 번호가 3-way 핸드셰이크 규칙대로 합의된다.
2. ESTABLISHED 상태에서 한쪽 `close()` 후 펌프하면 FIN/FIN_ACK 교환을 거쳐 **양쪽 모두 `CLOSED`** 가 되며, 수동측은 `PEER_CLOSED` 이벤트를 받는다.
3. `ManualClock`을 `heartbeat_interval` 이상 전진시킨 뒤 `pump()` 하면 HEARTBEAT가 송출되어 상대가 수신한다(real sleep 없이 가상 시간으로).
4. 하트비트가 끊긴 채 `idle_timeout`을 초과하도록 가상 시간을 전진시키면 `TIMED_OUT` 이벤트가 발생하고 상태가 `CLOSED`로 전이한다.
5. 세션 상태머신은 채널·클럭 인스턴스에 의존하지 않으며(시각은 `now` 인자 주입), 동일 입력 시퀀스에 대해 결정적으로 동작한다.
6. `examples/session_loopback.py` 실행 시 핸드셰이크 수립 → 하트비트 → graceful 종료(양쪽 CLOSED) 흐름이 출력된다.
7. `pytest`로 M1·M2 테스트가 전부 통과한다(M1 회귀 없음).
