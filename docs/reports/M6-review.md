# M6 리뷰보고서 (review)

## 비판점

### 차단 (0건)

차단 이슈 없음. 완료 기준 1~9 충족, 전체 테스트 139건 통과, 예제 6종 정상. **회고 최우선 미반영 후속이던
M4 flush-on-close가 앱 레벨 완료 핸드셰이크로 해소**됐다 — 송신측은 ACK(신뢰성 스트림)를 받은 뒤에만
종료하므로 전 청크 전달·SHA-256 검증이 보장된다. 이번 사이클도 소스 결함 발견 없음.

### 권장 (3건)

1. **파일 기본 stream_id가 채팅 기본(=1)과 동일** — `FILE_STREAM_ID = DEFAULT_STREAM_ID`. 인자로
   오버라이드 가능하나, **채팅과 동시 병행** 시 호출자가 다른 stream_id를 반드시 지정해야 한다(기본값만
   쓰면 두 앱이 같은 스트림에서 프레임이 섞임). 파일에 별도 기본 스트림 상수(예: 2 또는 `open_stream`)
   부여를 검토. 완료 기준 7("전용 스트림으로 채팅과 병행 가능")은 API로는 충족되나 기본값은 공유 상태.
2. **송신측 `progress`가 전달이 아닌 "큐잉"을 측정** — `_send_all_chunks_and_done`가 ACCEPT 직후 전
   청크를 ARQ에 **즉시 일괄 큐잉**하므로 `FileSender.progress`는 한 펌프 만에 ~1.0이 된다(전달/ACK 기준
   아님). 의미 있는 진행률은 **수신측 `progress`**(실제 받은 바이트). 송신 진행률이 필요하면 ARQ의
   누적 ACK 기반(전달된 바이트) 노출이 필요(현재 ARQ가 stream별 acked 바이트를 노출하지 않음).
3. **청크 일괄 큐잉 → 대용량 파일 시 메모리** — 전 청크를 ARQ `_pending` 버퍼에 한꺼번에 적재하므로
   파일 전체가 송신 ARQ 메모리에 상주한다. 인메모리 단일 파일(M6 범위)에선 무방하나, 대용량/스트리밍·
   재개(후속)에서는 **윈도우가 열릴 때 청크를 점진 공급**하는 흐름 제어가 필요.

### 사소 (2건)

4. **OFFER 필드 검증 느슨** — 수신측이 `meta.get("size"/"sha256")`를 타입 검증 없이 신뢰. 손상/누락
   OFFER면 `_expected_sha`가 None이 되어 DONE 검증에서 불일치→NACK로 안전 처리되나, 명시적 OFFER
   유효성 검사를 두면 더 견고.
5. **`auto_accept=False` 수동 수락 경로 미완성** — OFFER 메타만 저장하고 IDLE 유지(수동 accept 메서드
   없음). 범위 밖으로 docstring에 명시됨. 양방향/대화형 수락이 필요해지면 보강.

## 수정 내용

- 차단 0건이라 리뷰 단계의 소스 수정은 없음.

## 검증

- `python -m pytest -q` → **139 passed in 4.16s** (M1~M5 109 + M6 30, 회귀 없음).
- 예제 6종: `echo`·`session`·`reliable`·`chat`·`qr`·`file` loopback 전부 정상
  (`file_loopback`: loss=20% 링크에서 3072B 전송, 진행률 25/50/75/100%, byte+SHA-256 MATCH, 양쪽 CLOSED, ~0.27s).
- 통합 테스트가 무손실/손실 전송·완료 핸드셰이크·무결성 실패(NACK)·진행률·결정성·QR 채널 위 전송까지 커버.
- 잔여 리스크: 권장 1(스트림 공유)은 채팅+파일 동시 사용 시, 권장 3(메모리)은 대용량 파일에서 재검토 필요.
  현재 단방향 단일 파일·인메모리 범위에서는 문제 없음.

## 릴리즈 판정

**가능** — 추천 버전: **v0.6.0 (minor)**

- 완료 기준 1~9 전부 충족, 차단 이슈 0건. 파일 전송 앱(로드맵 5단계) 신규 추가 = 1.0 이전 minor.
  기반 v0.5.0(릴리즈 완료) → 목표 v0.6.0.
- 핵심 성과: **flush-on-close(회고 최우선 후속) 해소**. 사용자 최초 우선순위("기본 통신 → 파일 전송")의
  파일 전송이 완성됨.
- 권장 3건·사소 2건은 모두 후속(병행 데모/대용량/정리 마일스톤)이 집어갈 항목으로 릴리즈를 막지 않는다.
